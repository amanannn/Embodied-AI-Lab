# Vertical Applications

中文： [README.md](./README.md)

## Core Question

How do embodied AI systems become useful in real scenario-specific environments such as inspection or industrial workflows?

## Why This Direction Matters

Application directions force the curriculum to integrate perception, planning, control, and systems thinking under concrete operational constraints.

## Level Structure

- `level-1-python`: **Current main product**. Pure Python implementations with no ROS2 / Gazebo / Isaac / GPU dependencies, runnable in a Python 3.10+ runtime environment. Covers scenario-scoped application exercises.
- `level-2-ros2-bridge`: **Engineering bridge layer**. Connects Level 1 application exercises into ROS2 / C++ / real robot software stacks, targeting Ubuntu environments.
- `level-3-research`: **Research extension layer**. Domain-specific system design extensions.

## Representative Experiments

- `x01-autonomous-inspection-capstone`

## Suggested Entry Point

Enter this direction after completing at least one perception path and one control or navigation path.

## Research Extensions

Inspection, manufacturing, warehousing, and domain-specific embodied system design under operational constraints.
