## 一、什么是 Agentic RL

Agentic RL 是将**强化学习范式应用于大语言模型/多模态模型的训练**，使模型具备自主决策、工具使用、多步推理等 Agent 能力。

| 概念 | 说明 |
|------|------|
| **Agent** | 能感知环境、制定计划、使用工具、执行行动的自主智能体 |
| **RL** | 通过试错和奖励信号学习最优策略 |
| **Agentic RL** | 用 RL 训练模型成为更好的 Agent |

> 传统 RL 训练游戏 AI、机器人控制；Agentic RL 训练大模型学会工具调用、代码生成、多步推理、自我反思。

## 二、为什么需要 Agentic RL

### 2.1 监督学习的局限

| 问题 | 说明 |
|------|------|
| **分布偏移** | 训练数据中的"正确轨迹"未必覆盖所有真实场景 |
| **稀疏奖励** | Agent 任务的正确答案可能需要 10+ 步推理，中间步骤没有标注 |
| **探索需求** | Agent 需要尝试不同的行动序列，SFT 只能模仿已有的 |

### 2.2 RL 的优势

| RL 能力 | 对 Agent 的意义 |
|---------|---------------|
| **试错学习** | 从失败中学习，不依赖完美标注 |
| **长期规划** | 优化累积奖励而非单步正确率 |
| **探索** | 发现训练数据中没有的策略 |
| **自我改进** | Agent 生成轨迹 → 评估 → 迭代优化 |

## 三、核心技术路线

### 3.1 RLHF——从人类反馈中学习

```
Step 1: SFT —— 监督微调（人类示范）
Step 2: 训练奖励模型 —— 人类对多个模型回答排序
Step 3: PPO —— 用 RL 优化模型，最大化奖励模型给出的分数
```

| 组件 | 作用 |
|------|------|
| **Policy Model** | 正在训练的主模型 |
| **Reference Model** | 冻结的 SFT 模型——约束 Policy 不要跑太远（KL 惩罚） |
| **Reward Model** | 人类偏好训练的评分器 |
| **PPO** | 近端策略优化——稳定的 RL 算法 |

$$L_{PPO} = \mathbb{E}\left[r_\theta(x, y) - \beta \cdot D_{KL}(\pi_\theta \| \pi_{ref})\right]$$

> RLHF 是 ChatGPT 成功的核心技术——SFT 让模型"会说人话"，RLHF 让模型"说对的话"。

### 3.2 GRPO——群组相对策略优化

DeepSeek-R1 使用的核心算法：

| 特点 | 说明 |
|------|------|
| **无需奖励模型** | 用群组内多个回答的相对排名替代独立评分 |
| **减少训练开销** | 省去训练和维护奖励模型的计算成本 |
| **群组对比** | 同一 prompt 生成多个回答 → 好答案 vs 差答案 → 相对奖励 |

```python
# GRPO 伪代码
for prompt in batch:
    responses = model.generate(prompt, num_samples=K)  # 生成 K 个回答
    rewards = [rule_based_score(r) for r in responses]  # 基于规则评分
    advantages = (rewards - mean(rewards)) / std(rewards)  # 群组归一化
    # 好答案概率 ↑，差答案概率 ↓
    loss = -mean(advantages * log_prob(good_responses))
```

### 3.3 其他 Agentic RL 方法

| 方法 | 特点 | 代表 |
|------|------|------|
| **STaR** | 自举——用自己推理出的正确答案自我训练 | Zelikman et al., 2022 |
| **ReST** | 自我生成数据→过滤→SFT→RL 循环 | DeepMind, 2023 |
| **SPIN** | 自我对弈——当前版本 vs 上一版本 | Chen et al., 2024 |
| **RLVR** | 可验证奖励——数学题/代码题有明确对错 | DeepSeek-R1 |

## 四、奖励信号类型

| 奖励类型 | 适用场景 | 示例 |
|---------|---------|------|
| **人类偏好** | 开放式问答、写作 | "A 比 B 更好" |
| **规则验证** | 数学、代码、逻辑 | 答案是否匹配、代码是否通过测试 |
| **环境反馈** | 具身智能、游戏 | 是否成功抓取、是否通关 |
| **工具调用结果** | Agent 任务 | API 返回是否有效、文件是否正确读取 |
| **自我评估** | 通用推理 | 模型自评回答质量 |

> DeepSeek-R1 证明了：纯规则奖励（正确答案+格式规范）足以训练出强大的推理模型，不需要昂贵的人类标注。

## 五、Agentic RL 与具身智能

| 场景 | RL 如何用于具身 Agent |
|------|----------------------|
| **VLA 训练** | RL 优化机器人策略——抓取成功奖励 + 碰撞惩罚 |
| **任务规划** | 分解任务步骤的正确性作为奖励 |
| **Sim-to-Real** | 仿真中 RL 训练 → 迁移到真实机器人 |
| **自我改进** | 机器人执行→记录结果→RL 迭代 |
