## 一、从偃师造人到现代机器人

### 1.1 机器人简史

> "穆王惊视之，趋步俯仰，信人也。巧夫！领其颅，则歌合律；捧其手，则舞应节。"——《列子·汤问》

| 时期 | 里程碑 | 意义 |
|------|--------|------|
| 公元前4世纪 | 阿基塔斯蒸汽驱动的"鸽子"飞行器 | 最早的机械自驱动装置 |
| 1495年 | 达芬奇人形机器人草图 | 能坐起、挥臂、移动头部 |
| 1961年 | Unimate 工业机器人 | 机器人从"玩具"变成"工具" |
| 1973年 | KUKA FAMULUS（六轴） | 可编程自主运行 |
| 2000年 | 本田 ASIMO | 双足奔跑、上下楼梯 |
| 2013年 | 波士顿动力 Atlas | 极强的运动控制能力 |

### 1.2 工业机器人之后

工业机器人已经相对成熟。人们开始探索更多场景的机器人：

| 场景 | 代表 | 依赖技术 |
|------|------|---------|
| 医疗 | 微创手术机器人 | 视觉 + 精密控制 |
| 物流 | 仓储运输机器人 | 视觉 + SLAM |
| 服务 | 展厅导览机器人 | 视觉 + 语音 + NLP |
| 家庭 | 扫地机器人 | 视觉 + SLAM |
| 人形 | 智能人形机器人 | 全部技术栈 |

> 新的关注点不再是运动能力，而是**智能**。

## 二、什么是智能机器人？

### 2.1 两个核心维度

机器人 → 智能机器人 ≈ 人类，需要具备：

| 维度 | 定义 | 现状 |
|------|------|------|
| **自主能力** | 尽可能少的人类干预 | 部分场景已实现 |
| **泛化能力（通用能力）** | 新场景、新任务中自适应 | 仍存在巨大挑战 |

### 2.2 人工智能演进

| 时期 | 阶段 | 能力 |
|------|------|------|
| 1956-1960s | 符号推理 | 数学证明 |
| 1970-1980s | 专家系统 | 医疗/化学特定领域 |
| 1990s-2010 | 机器学习 | 处理实际问题 |
| 2011-2022 | 深度学习 | 图像/文本/语音 |
| **2022 至今** | **大模型** | **通用任务处理** |

> 目前最强的大模型 + 最先进的人形机器人 = 智能机器人？答案在"具身智能"中。

## 三、如何构建一个智能机器人？

### 3.1 硬件层

| 传感器 | 作用 |
|--------|------|
| RGB 相机 | 2D 视觉信号 |
| 3D 相机 / LiDAR | 3D 点云信号 |
| 触觉 / 力反馈 | 接触与力的感知 |
| 位姿传感器 | 本体感知 |
| 麦克风 | 语音信号 |

> 我们已经能造出具备基本性能的机器人硬件和高精度的传感器。

### 3.2 软件与算法层

| 层面 | 角色 | 功能 | 输入 → 输出 |
|------|------|------|------------|
| **大脑** | 具身感知 + 具身推理 | 综合分析环境信息与自身状态，做出决策和规划 | 传感器数据 → 决策/规划 |
| **小脑** | 具身执行 | 将规划转化为运动指令 | 规划结果 → 运动指令 |
| **下位机** | 运控执行 | 执行运动指令 | 指令 → 物理动作 |

```
传感器 → 大脑（感知 + 推理）→ 小脑（执行）→ 下位机（运控）
   ↑_______________________________________________↓
```

### 3.3 实例：清理咖啡

任务分解：
1. 扶正杯子并拿起杯盖
2. 找到抹布
3. 用抹布擦拭地面
4. 将抹布放回
5. 将杯子和杯盖扔掉

机器人需要：感知场景（杯子/咖啡/抹布）→ 推理步骤 → 生成轨迹 → 执行动作。

### 3.4 当前三大环节的成熟度

| 环节 | 成熟度 | 说明 |
|------|--------|------|
| **具身感知** | ⭐⭐⭐ | LLaVA 等多模态大模型已能做到多传感器综合分析 |
| **具身推理** | ⭐⭐ | 主流大模型在具身推理 benchmark 上表现不足 |
| **具身执行** | ⭐ | 最大瓶颈 |

#### 执行环节的两种方案对比

| 方案 | 成功率 | 流畅度 | 泛化能力 |
|------|--------|--------|---------|
| 多模态大模型生成指令 | 较低（60%-70%） | 不够流畅 | 物品泛化 |
| 扩散小模型 | 较高（90%以上） | 流畅 | 位置泛化或无泛化 |

> 生成技能库 API 或代码 API：现实场景过于复杂，构建完整技能库几乎不可能。因此，当前人工智能还不足以让机器人更智能——**需要具身智能**。

## 四、什么是具身智能？

### 4.1 核心实验：Held & Hein, 1963

| 组别 | 条件 | 结果 |
|------|------|------|
| **主动组** | 可以自由移动，眼中物体随动作变化 | **学会走路**，意识到"物体变大 = 靠近自己" |
| **被动组** | 被移动，只能看到物体变大/缩小 | 没有学会走路 |

> 有行走条件才能学会走路——有物理身体，可以进行交互。这就是"具身"的本质含义。

### 4.2 定义

| 层面 | 定义 |
|------|------|
| **定义 1** | 一种基于物理身体进行感知和行动的智能系统，通过智能体与环境的交互获取信息、理解问题、做出决策并实现行动 |
| **实质** | 强调有物理身体的智能体通过与物理环境进行交互而获得智能的研究范式 |

> 抽象的智能（围棋、文本）vs 具身智能（移开遮挡物后的识别）——有身体的智能需要与环境交互才能涌现。

### 4.3 具身智能系统架构

```
环境（真实环境 ↔ 仿真环境 ↔ Sim2Real）
    ↕
智能体
    ├── 感知（物/场景/人）
    ├── 推理（任务规划：基于规则 / 基于大模型）
    └── 执行（导航 / 具身问答 / 交互）
```

## 五、具身感知

### 5.1 四类感知对象

| 感知类型 | 内容 | 传感器 | 下游任务 |
|---------|------|--------|---------|
| **物体感知** | 几何形状、铰接结构、物理属性 | RGB + 3D 相机 | 位姿估计、抓取、可供性预测 |
| **场景感知** | 场景重建与理解 | RGB + 深度 | 主动探索、空间关系、时序变化 |
| **行为感知** | 手势、姿态、行为理解 | RGB + 触觉 | 社交导航、自动驾驶 |
| **表达感知** | 情感、意图 | 视觉 + 麦克风 | 人机协作 |

> 重点需要感知能力的机器人类型：服务机器人、人机协作用机器人、社交导航机器人、环境探索机器人。

### 5.2 感知流程

```
传感器（RGB/RGB-D/麦克风/触觉/热感）
    → 信息编码（表示/建模/重建）
        → 信息理解
            → 下游任务
```

### 5.3 3D 数据表示与编码

| 数据格式 | 描述 | 来源 | 编码方法 |
|---------|------|------|---------|
| **点云** | 一组 (x,y,z) 点 + 特征 | LiDAR | PointNet, PointNet++ |
| **网格（Mesh）** | 点、线、面表示物体表面 | CAD、点云转换 | MeshNet |
| **体素（Voxel）** | 立方体网格 | 点云转换 | VoxelNet, DeepSDF, Occupancy Network |
| **深度图** | 像素对应的深度值 | 双目/结构光/ToF | GVCNN |

### 5.4 3D 编码方法详解

| 方法 | 核心思想 | 论文 |
|------|---------|------|
| **PointNet** | MLP 编码每个点 → 聚合为全局特征 | Qi et al., CVPR 2017 |
| **PointNet++** | 层次化采样 + 局部特征捕捉（PointNet 改进版） | Qi et al., NIPS 2017 |
| **MeshNet** | MLP+CNN 编码每个面的空间和结构特征 | Feng et al., AAAI 2019 |
| **VoxelNet** | 体素化 + MLP+CNN 编码 | Zhou et al., CVPR 2018 |
| **DeepSDF** | 神经网络拟合连续 SDF 函数：体素坐标 → 离表面的距离 | Park et al., CVPR 2019 |
| **Occupancy Network** | 神经网络拟合体素出现概率函数 | Mescheder et al., CVPR 2019 |

> PointNet、MeshNet、VoxelNet 将 3D 数据视为**离散单元**进行卷积编码（类似 CV 中对 2D 图片）。DeepSDF 和 Occupancy Network 则用**连续函数**来表示物体形状——不局限于单个物体，引入 Code 作为形状标签。

### 5.5 几何感知的下游：物体位姿估计

位姿估计：预测物体在 3D 空间中的 6 自由度位姿（3 平移 + 3 旋转）。

| 类型 | 是否需要 CAD 模型 | 方法 |
|------|------------------|------|
| **实例级** | 需要 | 已知中心点和初始朝向 → 估计当前位姿 |
| **类别级** | 不需要 | 通过训练集中同类物体的平均中心点和朝向进行"预设" |

> NOCS（Normalized Object Coordinate Space）：在每个像素上预测标准空间坐标 (x,y,z)，结合深度得到平移和朝向。

### 5.6 感知研究与具身智能的关系

> 正如句法分析、词性标注之于早期 NLP，物体感知是具身智能的"中间任务"——助力下游推理和执行，满足实际应用需要。

- 有观点认为**显式世界模型**是 AI 的后续方向，此时感知具有更重要意义
- 3D 数据处理方式与 2D 图像处理高度相似，大模型方法论可迁移至 3D

## 六、当前瓶颈与展望

### 6.1 三环节现状

| 环节 | 状态 | 关键挑战 |
|------|------|---------|
| 具身感知 | 相对成熟 | 复杂场景中的实时性与鲁棒性 |
| 具身推理 | 发展中 | 大模型成功率不高，泛化有限 |
| 具身执行 | 最大瓶颈 | 技能库不完备，端到端轨迹生成质量不足 |

### 6.2 核心矛盾

1. **训练数据稀缺**：机器人数据远少于文本/图像数据——Open X-Embodiment 试图通过开放平台缓解
2. **跨本体泛化**：不同形态的机器人需要不同的动作空间
3. **Sim2Real Gap**：仿真中训练的策略在真实环境中的迁移
4. **实时性要求**：感知→推理→执行闭环需要低延迟

## 七、总结

```text
具身智能 = 有身体 + 能交互 + 会学习

                      ┌── 物体感知（几何/铰接/物理）
                      ├── 场景感知（重建/理解）
          ┌── 具身感知 ─┤
          │             ├── 行为感知（手势/姿态/行为）
          │             └── 表达感知（情感/意图）
          │
具身智能 ─┼── 具身推理 ── 任务规划（基于规则 / 基于大模型）
          │
          └── 具身执行 ── 导航 / 操作 / 交互（API / 关节角度 / 代码）
```

> 大模型时代的具身智能，不是简单地把 GPT 装进机器人。感知相对成熟，推理正在追赶，执行是最大瓶颈。3D 大模型可能是下一个突破口。
