## 一、从机器人到智能机器人

### 1.1 什么是智能机器人？

从偃师造人至今，机器人经历了从机械体到智能体的演化。智能机器人与传统机器人的关键区别有两个维度：

| 维度 | 传统机器人 | 智能机器人 |
|------|-----------|-----------|
| **自主能力** | 预编程，固定流程 | 感知环境 → 自主决策 → 执行 |
| **泛化能力** | 单一场景，换环境即失效 | 新场景、新任务中自适应 |

> 人工智能真的让机器人智能了吗？硬件、传感器、运控技术相对成熟，但软件算法层面——具身感知、推理、执行——仍存在巨大挑战。

### 1.2 构建智能机器人的软硬件架构

| 层面 | 功能 | 输入 | 输出 |
|------|------|------|------|
| **硬件层** | 感知与执行基础 | 2D视觉/3D点云/触觉/力反馈/位姿/语音 | — |
| **大脑** | 具身感知 + 具身推理 | 多传感器信息 + 自身状态 | 决策与运动规划 |
| **小脑** | 具身执行 | 运动指令 | 下位机运控（代码/技能库API/关节角度） |

```
硬件感知 → 大脑(感知+推理) → 小脑(执行) → 下位机运控
    ↑_______________________________________________↓
                      闭环反馈
```

## 二、视觉基础模型演进：从像素到语义

### 2.1 早期视觉：机器首先看到的是像素

早期计算机视觉的起点是"可计算"而非"可理解"——摄像机将连续世界采样为离散像素矩阵，算法处理的是灰度值、颜色和梯度。

| 阶段 | 方法 | 解决什么 |
|------|------|---------|
| **图像处理** | 滤波、增强、去噪 | 降低噪声，突出结构 |
| **边缘检测** | Canny、Sobel | 从灰度突变中找轮廓 |
| **角点检测** | Harris | 建立图像间的局部对应 |
| **人工特征** | SIFT、HOG、ORB | 人设计规则，机器执行提取 |

> 传统视觉的能力上限：能匹配，难理解。光照、姿态、遮挡变化都会破坏人工设计好的规则。

### 2.2 统计学习时代：特征提取 + 分类器

| 阶段 | 作用 | 典型方法 |
|------|------|---------|
| 预处理 | 归一化、灰度化 | 像素操作 |
| 特征提取 | 图像→可比较的向量 | SIFT / HOG / LBP |
| 分类判断 | 根据特征向量输出类别 | SVM / Boosting / Random Forest |
| 后处理 | 修正边界、过滤噪声 | NMS / 阈值筛选 |

> 这一阶段系统能力依赖特征质量——特征是人工设计的，识别上限被人为设定的规则锁死。

### 2.3 深度学习时代：特征开始由模型自己学习

CNN 的核心变革：**特征不再写在规则里，而是存储在模型参数中**。

#### CNN 表征的层级性

| 层级 | 提取内容 | 作用 |
|------|---------|------|
| **低层** | 边缘、颜色、纹理 | 局部视觉线索 |
| **中层** | 局部形状、部件、重复模式 | 组合为结构 |
| **高层** | 物体类别、场景概念 | 语义理解 |

#### 里程碑

| 模型 | 年份 | 核心贡献 |
|------|------|---------|
| CNN (LeNet/AlexNet) | 1998/2012 | 端到端训练的特征提取 |
| VGG | 2014 | 深层 + 小卷积核标准化 |
| **ResNet** | 2015 | 残差连接——深层网络可稳定训练 |
| Backbone + Head | — | 通用骨干 + 任务头，特征复用思想雏形 |

### 2.4 视觉任务扩展：从分类到结构化输出

| 任务 | 输出粒度 | 回答 | 代表方法 |
|------|---------|------|---------|
| **分类** | 图像级 | "图中是什么？" | ResNet, ViT |
| **检测** | 区域级 | "物体在哪里？" | R-CNN, YOLO, DETR |
| **分割** | 像素级 | "边界在哪里？" | FCN, Mask R-CNN, SAM |

> 对机器人而言，Mask 比检测框更接近可抓取、可投影的操作区域。

### 2.5 任务模型的局限：强能力但弱通用

CNN 时代的视觉模型很强，但多数仍是**任务专用**：

| 局限 | 表现 |
|------|------|
| 封闭类别 | 只能识别训练集定义好的类别 |
| 换任务重训练 | 分类/检测/分割需要不同 head |
| 语言交互弱 | 无法直接接受自然语言输入 |

## 三、视觉基础模型：通用视觉底座

### 3.1 新范式：预训练 → 多任务迁移

视觉模型从"为某个任务训练"转向"先训练通用模型，再迁移到多任务"。

### 3.2 Transformer 的启发

2017年 "Attention Is All You Need" 提供了一种跨模态复用的建模语言：**Token + Attention**。

### 3.3 ViT：图像也可以被切成 Token

| 步骤 | 操作 |
|------|------|
| 1. Patch | 图像切为固定大小的小块 |
| 2. Visual Token | 每个 patch 映射为一个视觉 token |
| 3. Self-Attention | Transformer 建模所有 token 间的全局关系 |

> ViT 让视觉进入了与语言模型相似的架构体系，为多模态统一建模奠定了基础。

### 3.4 自监督学习：摆脱标注依赖

| 方法 | 核心思想 |
|------|---------|
| **对比学习** | 相似图像靠近，不同图像远离 |
| **遮挡重建（MAE）** | 遮住部分，从局部推断整体 |
| **自蒸馏（DINO）** | 教师-学生机制学习稳定表示 |

## 四、开放词汇感知：让视觉听懂自然语言

### 4.1 从封闭类别到开放词汇

传统模型依赖固定类别表。真实机器人任务中，用户不会按数据集类别表下命令，而是用自然语言：

> "拿起桌面左侧的红色杯子"

| 传统 | 开放词汇 |
|------|---------|
| 类别标签："cup" | 类别 + 属性 + 位置 + 任务 |
| 封闭集合 | 自然语言查询 |

### 4.2 CLIP：让图像和语言进入同一空间

CLIP 的核心创新：**图文双塔 + 对比学习**。

| 组件 | 功能 |
|------|------|
| 图像编码器 | 图片 → 视觉向量 |
| 文本编码器 | Prompt → 语言向量 |
| 共同语义空间 | 正确图文对靠近，错误图文对远离 |

#### CLIP 的能力与边界

| ✅ 能做 | ❌ 不能直接做 |
|---------|-------------|
| 图文检索 | 精确目标框 |
| Zero-shot 分类 | 像素级 Mask |
| 语义匹配 | 机器人可操作边界 |

> CLIP 让视觉听懂语言，但还没告诉机器人目标精确在哪里。这对具身智能而言不够。

### 4.3 Grounding DINO：文本到区域框的桥梁

CLIP 输出全图语义评分，Grounding DINO 进一步输出**与文本相关的二维目标框**。

| 组件 | 作用 |
|------|------|
| Image Backbone | 提取多尺度视觉特征 |
| Text Encoder | 将 Prompt 编码为文本 token |
| **Feature Enhancer** | 文本 token ↔ 图像区域双向注意力交互 |
| Decoder | 输出与文本语义匹配的目标框 |

```
输入：图像 + "red cup on the left"
  → Feature Enhancer（文本约束注入视觉区域）
    → Decoder → 输出候选框 + 文本匹配分数
```

> Grounding DINO 的意义：**"语言中的目标"第一次具有了图像坐标**。

### 4.4 SAM：从框到像素级分割

SAM（Segment Anything Model）提供通用分割能力。与 Grounding DINO 配合：

```
自然语言 → Grounding DINO → 候选框 → SAM → 像素级 Mask → 机器人可操作区域
```

这构成了开放词汇感知的完整技术链路。

## 五、具身智能任务链

机器人完成任务需要形成**从感知到执行的闭环**：

| 阶段 | 功能 | 输入 | 输出 |
|------|------|------|------|
| **01 感知** | 获取环境状态 | 相机/深度/触觉 | 图像/深度/接触信息 |
| **02 理解** | 识别语义与关系 | 感知数据 | 目标/属性/场景关系 |
| **03 定位** | 确定目标位置 | 理解结果 | 框/Mask/位姿候选 |
| **04 规划** | 拆解任务为步骤 | 定位信息 | 子任务/路径/策略 |
| **05 执行** | 控制执行动作 | 规划步骤 | 轨迹/控制命令 |
| **06 反馈** | 闭环修正 | 执行结果 | 误差/状态更新 |

```
感知 → 理解 → 定位 → 规划 → 执行 → 反馈 → (回到感知)
```

## 六、具身感知：四类感知对象

| 感知类型 | 内容 | 关键技术 | 典型场景 |
|---------|------|---------|---------|
| **物体感知** | 几何形状、铰接结构、物理属性 | PointNet++, 3D 检测 | 抓取、操作 |
| **场景感知** | 场景重建与理解 | NeRF, 3D GS, 语义 SLAM | 导航、探索 |
| **行为感知** | 手势、人体姿态、行为理解 | Pose Estimation, Action Recognition | 人机协作 |
| **表达感知** | 情感、意图检测 | Facial/Audio Analysis | 服务机器人 |

### 6.1 3D 数据表示

| 数据格式 | 描述 | 来源 | 编码方法 |
|---------|------|------|---------|
| **点云** | 一组 (x,y,z) 坐标点 + 特征 | LiDAR | PointNet, PointNet++ |
| **网格（Mesh）** | 点、线、面表示物体表面 | CAD、点云转换 | MeshNet |
| **体素（Voxel）** | 立方体网格 | 点云转换 | VoxelNet, DeepSDF |
| **深度图** | 每个像素对应深度值 | 双目/结构光/ToF | GVCNN |

> PointNet 是点云编码的经典方法——用 MLP 编码每个点，再聚合全局特征。PointNet++ 针对其难以捕捉局部特征的缺点引入层次化采样。

## 八、当前瓶颈与前沿方向

### 8.1 具身智能当前瓶颈

| 瓶颈 | 表现 |
|------|------|
| **感知已较强** | LLaVA 等多模态大模型已能做到多传感器综合分析 |
| **推理仍有差距** | 主流大模型在具身推理 benchmark 上表现不足 |
| **执行有根本困难** | 场景太复杂，构建完整技能库几乎不可能 |

> 因此，当前人工智能还不足以让机器人真正智能——**需要具身智能**。

### 8.2 前沿研究方向

| 方向 | 说明 | 代表工作 |
|------|------|---------|
| **VLA 模型** | 视觉-语言-动作端到端模型 | Uni-Sight, RT-2 |
| **VLN 模型** | 视觉-语言-导航 | — |
| **多智能体** | 自动编程、复杂任务解译 | Hermes, OpenClaw |
| **轻量化** | 大模型/具身智能的边缘部署 | 量化/剪枝/蒸馏 |
| **预训练→微调→后训练** | 多模态大模型全流程 | — |

## 九、总结

```text
视觉模型演进：
  像素 → 手工特征 → CNN → ViT/Transformer → 视觉基础模型
                                           ↓
                                   CLIP（图文对齐）
                                           ↓
                               Grounding DINO（开放检测）
                                           ↓
                                   SAM（通用分割）
                                           ↓
                              具身智能 VLA（感知→推理→执行）

具身智能 = 视觉基础模型 + 语言理解 + 机器人执行 + 闭环反馈
```

> 大模型时代的具身智能，不是把大模型装进机器人那么简单。它需要视觉基础模型理解环境，需要语言模型理解任务，需要 VLA 模型连接感知与动作，还需要闭环反馈保证鲁棒性。这是一个系统工程。
