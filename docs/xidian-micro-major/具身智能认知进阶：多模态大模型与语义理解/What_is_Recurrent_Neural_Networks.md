## 一、为什么需要 RNN

传统的全连接网络和 CNN 都假设输入之间是**独立的**——每张图片、每个样本单独处理，前后没有关联。但很多任务天然具有**时序依赖**：

| 任务 | 输入 | 时序关系 |
|------|------|---------|
| 语言模型 | 文本序列 | "今天" → "天气" → "很好" 有先后 |
| 语音识别 | 音频帧 | 前后帧连续，不能乱序 |
| 视频理解 | 帧序列 | 动作随时间推进 |
| 时间序列预测 | 历史序列 | 过去值影响未来 |

> RNN 通过**带自反馈的神经元**，能够处理任意长度的时序数据——这是从"静态感知"迈向"动态理解"的关键一步。

## 二、RNN 的基本结构

### 2.1 宏观结构

```
输入层 → 隐藏层（含延迟器） → 输出层
              ↑_____↓
              反馈连接
```

| 组件 | 功能 | 类似 CNN |
|------|------|---------|
| **输入层** | 接收当前时刻的输入 $x_t$ | FC 输入 |
| **隐藏层** | 存储**活性值状态**（历史信息） | 特征图 |
| **延迟器** | 将上一时刻的隐藏状态传入当前 | ❌ CNN 没有 |
| **输出层** | 产生当前时刻的输出 $h_t$ | FC 输出 |

### 2.2 按时间展开

RNN 的核心——**同一个网络权重在不同时刻重复使用**：

```text
t=1        t=2        t=3        t=4
x₁ →┌─┐   x₂ →┌─┐   x₃ →┌─┐   x₄ →┌─┐
    │RNN│─h₁─→│RNN│─h₂─→│RNN│─h₃─→│RNN│─h₄→ ...
    └───┘     └───┘     └───┘     └───┘
```

$$h_t = f(W_{xh} x_t + W_{hh} h_{t-1} + b_h)$$

| 符号 | 含义 | 形状 |
|------|------|------|
| $x_t$ | $t$ 时刻的输入 | $(d_{in}, 1)$ |
| $h_{t-1}$ | $t-1$ 时刻的隐藏状态 | $(d_h, 1)$ |
| $W_{xh}$ | 输入 → 隐藏的权重 | $(d_h, d_{in})$ |
| $W_{hh}$ | 隐藏 → 隐藏的权重（时序连接） | $(d_h, d_h)$ |
| $h_t$ | $t$ 时刻的隐藏状态（输出） | $(d_h, 1)$ |

> 关键洞察：**$W_{xh}$ 和 $W_{hh}$ 在所有时刻共享**——无论序列多长，参数量不变。这使得 RNN 可以处理变长序列。

## 三、RNN 的前向传播

```python
# RNN 的单步前向传播
def rnn_step(x_t, h_prev, W_xh, W_hh, b_h):
    """
    x_t:    当前输入 (d_in,)
    h_prev: 上一时刻隐藏状态 (d_h,)
    """
    # 输入变换 + 历史状态变换
    h_t = torch.tanh(W_xh @ x_t + W_hh @ h_prev + b_h)
    return h_t

# 整个序列的前向传播
def rnn_forward(X, h0, W_xh, W_hh, b_h):
    """
    X: 输入序列 (seq_len, batch, d_in)
    返回所有时刻的隐藏状态
    """
    h = h0
    outputs = []
    for t in range(len(X)):
        h = rnn_step(X[t], h, W_xh, W_hh, b_h)
        outputs.append(h)
    return torch.stack(outputs)
```

## 四、RNN 的反向传播：BPTT

### 4.1 为什么叫"穿越时间的反向传播"

BPTT（Backpropagation Through Time）——将 RNN 按时间展开后，它就像一个多层前馈网络，层数等于序列长度。

$$h_t = \tanh(W_{xh}x_t + W_{hh}h_{t-1} + b)$$

对 $W_{hh}$ 求梯度时，梯度需要**从最后一个时刻回传到第一个时刻**：

$$\frac{\partial L}{\partial W_{hh}} = \sum_{t=1}^{T} \frac{\partial L_t}{\partial h_t} \frac{\partial h_t}{\partial W_{hh}}$$

而 $\frac{\partial h_t}{\partial W_{hh}}$ 又依赖于 $h_{t-1}$，$h_{t-1}$ 又依赖于 $h_{t-2}$……形成长链。

### 4.2 梯度消失与梯度爆炸

| 问题 | 原因 | 表现 | 解决 |
|------|------|------|------|
| **梯度消失** | 激活函数梯度 < 1（如 tanh），链式相乘趋近 0 | 早期时刻学不到任何东西 | LSTM、GRU、梯度裁剪 |
| **梯度爆炸** | 权重过大，梯度链式相乘指数增长 | Loss → NaN | 梯度裁剪（Gradient Clipping） |

```python
# 梯度裁剪——最简单的防爆炸手段
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

> RNN 理论上能处理任意长序列，但实践中受限于梯度消失，普通 RNN 的有效记忆大约只有 10-20 步。

## 五、PyTorch 中的 RNN

```python
import torch.nn as nn

# 内置 RNN 层
rnn = nn.RNN(
    input_size=128,      # 输入特征维度 d_in
    hidden_size=256,     # 隐藏层维度 d_h
    num_layers=2,        # 堆叠层数
    batch_first=True,    # (batch, seq, feature) 格式
    bidirectional=False  # 是否双向
)

# 前向传播
x = torch.randn(32, 50, 128)   # (batch, seq_len, input_size)
output, h_n = rnn(x)            # output: (32, 50, 256)
                                # h_n:    (2, 32, 256) — 最后一层的隐藏状态
```

| 参数 | 说明 |
|------|------|
| `input_size` | 每个时刻输入的向量维度 |
| `hidden_size` | 隐藏状态向量的维度 |
| `num_layers` | 堆叠的 RNN 层数 |
| `bidirectional` | 双向 RNN——同时从前向和后向处理序列 |
| `batch_first` | 推荐设为 True |

## 六、RNN 的局限性

> 当序列较长时，普通 RNN 会出现梯度消失，丢失远距离依赖关系

### 6.1 局限性总结

| 局限 | 原因 | 影响 |
|------|------|------|
| 短时记忆 | 梯度消失 | 只能记住约 10-20 步的上下文 |
| 串行计算 | $h_t$ 依赖 $h_{t-1}$ | 无法并行，训练慢 |
| 容量限制 | 单一隐藏状态 $h$ | 难以同时存储多种信息 |
| 无选择性 | 每个时刻同等对待所有输入 | 无法"忘记"无关信息 |

### 6.2 解决方案

| 方案 | 核心思想 | 代表 |
|------|---------|------|
| LSTM | 门控机制 + 细胞状态 | 选择性记忆/遗忘 |
| GRU | 简化版 LSTM（两个门） | 参数更少，效果相近 |
| 双向 RNN | 同时看前后文 | BiLSTM |
| 注意力机制 | 直接连接任意距离 | Transformer 取代了 RNN |

## 七、RNN 的应用场景

| 场景 | 输入 | 输出 | 典型架构 |
|------|------|------|---------|
| 文本分类 | 词序列 | 类别 | RNN → 最后时刻输出 → FC |
| 序列标注（NER） | 词序列 | 每个词的标签 | RNN → 每时刻输出 → FC |
| 机器翻译 | 源语言序列 | 目标语言序列 | Encoder RNN → Decoder RNN |
| 时间序列预测 | 历史值序列 | 未来值 | RNN → FC |
| 语音识别 | 音频特征序列 | 文本序列 | 多层 BiLSTM |

---

## 八、RNN vs CNN vs Transformer

| 维度 | RNN | CNN | Transformer |
|------|-----|-----|-------------|
| 时序建模 | ✅ 天然适合 | ⚠️ 需 1D 卷积 | ✅ 位置编码 |
| 长程依赖 | ❌ 梯度消失 | ⚠️ 需多层 | ✅ 自注意力 |
| 并行计算 | ❌ 串行 | ✅ 并行 | ✅ 并行 |
| 参数量 | 少（共享权重） | 少 | 多（QKV 投影） |
| 训练速度 | 慢 | 快 | 快（短序列） |
