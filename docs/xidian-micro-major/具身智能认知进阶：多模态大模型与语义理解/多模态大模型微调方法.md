## 二、全量微调

更新模型**全部参数**——上限最高，代价最大。

| 模型大小 | 参数显存（FP16） | 训练显存（含优化器+梯度） | 最低 GPU |
|---------|----------------|----------------------|---------|
| 1B | ~2 GB | ~10 GB | RTX 3090 (24G) |
| 7B | ~14 GB | ~60 GB | 2×A6000 (48G) |
| 13B | ~26 GB | ~110 GB | 4×A100 (80G) |
| 70B | ~140 GB | ~560 GB | 8×A100 (80G) |

> 全量微调 7B 需要约 60GB 显存——远超单张消费级显卡。这就是 PEFT 存在的理由。

## 三、LoRA——低秩适配

### 3.1 核心数学

预训练权重 $W_0 \in \mathbb{R}^{d \times k}$ 的更新被约束为低秩分解：

$$W = W_0 + \Delta W = W_0 + BA$$

其中 $B \in \mathbb{R}^{d \times r}$, $A \in \mathbb{R}^{r \times k}$, $r \ll \min(d, k)$。

前向传播变为：

$$h = W_0 x + BAx = W_0 x + \frac{\alpha}{r} \cdot BAx$$

| 参数 | 含义 | 典型值 | 调参建议 |
|------|------|--------|---------|
| **r（rank）** | 低秩维度——决定可训练参数 | 8（省）/ 64（精度优先） | 简单任务 r=8 够；复杂多任务 r=64 |
| **α（alpha）** | 缩放因子 | r 的 2~4 倍 | α=2r 是常用起点 |
| **target_modules** | 哪些层加 LoRA | `q_proj,v_proj`（最少）→ 全注意力层 | 越多精度越高、参数也越多 |
| **lora_dropout** | 正则化 | 0.05~0.1 | 数据少时加大 |

### 3.2 为什么 LoRA 有效？

预训练权重矩阵 $W$ 虽然维度大，但**权重更新矩阵 $\Delta W$ 往往具有低"本征秩"**——模型微调时不需在全空间中搜索，只需沿少数几个重要方向调整。Aghajanyan et al. (2021) 证明了这一点。

### 3.3 完整微调脚本

```python
from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments, Trainer
from peft import LoraConfig, get_peft_model, TaskType
from datasets import load_dataset
import torch

# 1. 加载模型
model_name = "Qwen/Qwen2-7B-Instruct"
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype=torch.bfloat16,
    device_map="auto",
    trust_remote_code=True,
)
tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)
tokenizer.pad_token = tokenizer.eos_token

# 2. 配置 LoRA
lora_config = LoraConfig(
    r=16,                            # rank
    lora_alpha=32,                   # 缩放
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                    "gate_proj", "up_proj", "down_proj"],  # 全注意力 + FFN
    lora_dropout=0.05,
    bias="none",
    task_type=TaskType.CAUSAL_LM,
)
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# 输出: trainable params: 41,943,040 || all params: 7,615,610,880 || trainable%: 0.55%

# 3. 准备数据
def format_instruction(example):
    return {"text": f"<|user|>\n{example['instruction']}\n<|assistant|>\n{example['output']}"}

dataset = load_dataset("yahma/alpaca-cleaned")
dataset = dataset.map(format_instruction)

# 4. 训练
trainer = Trainer(
    model=model,
    args=TrainingArguments(
        output_dir="./qwen-lora-alpaca",
        per_device_train_batch_size=4,
        gradient_accumulation_steps=4,    # 等效 batch_size=16
        learning_rate=2e-4,
        num_train_epochs=3,
        logging_steps=10,
        save_strategy="epoch",
        bf16=True,                        # A100/H100 用 bf16
        gradient_checkpointing=True,      # 显存换速度
    ),
    train_dataset=dataset["train"],
)
trainer.train()

# 5. 保存——只存适配器（几 MB）
model.save_pretrained("./qwen-lora-alpaca-adapter")
```

## 四、QLoRA——量化微调

### 4.1 四重优化技术

| 技术 | 作用 | 显存节省 |
|------|------|---------|
| **4-bit NormalFloat** | 信息论最优的 4-bit 量化 | ~4× |
| **双量化** | 对量化常数再做一次量化 | ~0.4 bit/参数 |
| **分页优化器** | OOM 时自动将优化器状态换出到 CPU | 峰值降 30% |
| **LoRA** | 只训练低秩适配器 | ~100×（参数量） |

### 4.2 QLoRA 微调脚本

```python
from transformers import BitsAndBytesConfig

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",           # NormalFloat4
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,      # 双量化
)

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-70b-chat-hf",
    quantization_config=bnb_config,
    device_map="auto",
    trust_remote_code=True,
)
# 70B 模型 → 约 35 GB → 单卡 A6000 (48G) 即可！
```

### 4.3 显存全景对比

| 方法 | 7B | 13B | 34B | 70B | 405B |
|------|-----|-----|-----|-----|------|
| 全量微调 | ~60 GB | ~110 GB | 不可行 | 不可行 | 不可行 |
| LoRA (FP16) | ~16 GB | ~30 GB | ~80 GB | ~160 GB | 不可行 |
| QLoRA (4bit) | ~6 GB | ~10 GB | ~18 GB | ~35 GB | ~80 GB |

## 五、LoRA 变体

| 变体 | 改进点 | 使用场景 |
|------|--------|---------|
| **LoRA+** | 为 $B$ 和 $A$ 设置不同学习率 | 通用 |
| **AdaLoRA** | 自适应分配各层的 rank | 追求极致参数效率 |
| **DoRA** | 将权重分解为大小+方向，只微调方向 | 接近全量微调效果 |
| **QLoRA** | LoRA + 4bit 量化 | 消费级显卡微调大模型 |
| **LoRA-FA** | 冻结 $A$，只训练 $B$ | 进一步减半参数量 |

## 六、指令微调数据准备

### 6.1 数据格式

```json
[
  {
    "instruction": "给以下代码添加注释",
    "input": "def fib(n):\n    if n<=1: return n\n    return fib(n-1)+fib(n-2)",
    "output": "def fib(n):\n    \"\"\"计算第 n 个斐波那契数\"\"\"\n    if n <= 1:\n        return n\n    return fib(n - 1) + fib(n - 2)"
  }
]
```

### 6.2 主流数据集

| 数据集 | 规模 | 领域 | 质量 |
|--------|------|------|------|
| Alpaca | 52K | 通用 | 中——GPT-3 生成，有噪声 |
| Dolly | 15K | 通用 | 高——人工编写 |
| OpenOrca | 1M+ | 通用 | 中——GPT-4 生成 |
| UltraChat | 1.5M | 对话 | 中高 |
| ShareGPT | 90K | 对话 | 高——真实用户对话 |

### 6.3 视觉-语言微调

| 策略 | 操作 | 何时用 |
|------|------|--------|
| **LoRA LLM only** | 冻结 ViT + Projector → LoRA 微调 LLM | 保持视觉能力，提升对话 |
| **LoRA LLM + 解冻 Projector** | Projector 全量 + LLM LoRA | 适配新视觉编码器 |
| **LoRA LLM + 解冻 ViT 最后层** | 轻微调整视觉特征 | 视觉域偏移（医疗/遥感） |
| **全量** | 一切可训练 | 数据很多 + GPU 很足 |

## 七、微调实战 Checklist

- [ ] 确认基座模型和任务匹配——不要用纯文本模型做多模态微调
- [ ] 数据格式对齐——instruction/input/output 字段名与模型预训练一致
- [ ] 先做 LoRA 快速验证（r=8）——确认 loss 下降再加大 rank
- [ ] 监控 train loss + eval loss——loss 不降 = 学习率/数据有问题
- [ ] QLoRA 时注意——解冻的层不能是 4-bit（需要保持精度）
- [ ] 保存 adapter，不要存完整模型——几 MB vs 几十 GB
- [ ] 微调完做定性测试——用 5-10 条真实 case 手测效果
- [ ] 记录关键参数——r, α, lr, batch_size, epoch——方便复现
