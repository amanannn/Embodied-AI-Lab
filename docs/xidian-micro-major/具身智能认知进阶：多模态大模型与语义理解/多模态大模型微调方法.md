## 一、微调：从通用到专用

预训练让模型拥有通用知识，微调让模型适配特定任务。

| 阶段 | 数据量 | 目标 | 更新参数 |
|------|--------|------|---------|
| **预训练** | TB 级无标注数据 | 通用表征 | 全量 |
| **微调** | KB~MB 级标注数据 | 特定任务适配 | 视方法而定 |

---

## 二、全量微调

更新模型**全部参数**，最直接但最贵。

| 维度 | 说明 |
|------|------|
| **参数量** | 7B 模型 → 约 28 GB 显存（仅模型参数） |
| **训练显存** | 全量微调需 4× 模型大小 → 7B 需 ~100+ GB |
| **适用** | 有充足 GPU、数据差异大 |

> 全量微调精度上限最高，但 7B 模型一张 A100 都跑不了。

---

## 三、参数高效微调（PEFT）

### 3.1 为什么需要 PEFT

| 问题 | PEFT 方案 |
|------|----------|
| 全量 7B → 8×A100 | LoRA 7B → 一张 RTX 3090 |
| 每个任务存 14 GB | PEFT 适配器 ~80 MB |
| 灾难性遗忘 | 冻结主模型，不易遗忘 |

### 3.2 LoRA——低秩适配

权重更新矩阵用两个小矩阵的乘积近似：

$$W' = W + \Delta W = W + B \cdot A$$

其中 $B \in \mathbb{R}^{d \times r}$，$A \in \mathbb{R}^{r \times d}$，$r \ll d$。

| 参数 | 含义 | 典型值 |
|------|------|--------|
| **r（rank）** | 低秩维度 | 8, 16, 64 |
| **α（alpha）** | 缩放因子 | 16, 32 |
| **target_modules** | 加 LoRA 的层 | `q_proj`, `v_proj` |

```python
from peft import LoraConfig, get_peft_model

lora_config = LoraConfig(
    r=16, lora_alpha=32,
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.1, task_type="CAUSAL_LM",
)
model = get_peft_model(base_model, lora_config)
# 7B 模型 → 可训练参数仅 ~40M（0.6%）
```

### 3.3 QLoRA——量化 + LoRA

在 4-bit 量化模型上做 LoRA，消费级显卡微调 70B。

| 方法 | 7B 显存 | 70B 显存 |
|------|--------|---------|
| 全量微调 | ~100 GB | 不可行 |
| LoRA (FP16) | ~16 GB | ~160 GB |
| **QLoRA (4bit)** | **~6 GB** | **~35 GB** |

### 3.4 其他 PEFT 方法

| 方法 | 核心思路 | 特点 |
|------|---------|------|
| **Adapter** | Transformer 层间插入小型模块 | 推理有额外延迟 |
| **Prefix Tuning** | 输入前添加可训练虚拟 token | 适合生成任务 |
| **Prompt Tuning** | 只训练软 prompt 向量 | 参数最少 |
| **IA³** | 激活值乘以可学习向量 | 仅几千参数 |

## 四、指令微调

预训练模型会续写但不会"回答问题"。指令微调教会模型**指令→回答**格式。

```json
{
  "instruction": "用一句话解释什么是黑洞",
  "input": "",
  "output": "黑洞是恒星坍缩形成的时空区域，引力强到连光都无法逃逸。"
}
```

| 数据集 | 规模 | 特点 |
|--------|------|------|
| FLAN | 1.8K 任务 | 覆盖翻译/问答/推理 |
| Self-Instruct | GPT-4 生成 | 大模型生成指令数据 |
| Alpaca | 52K | 经典入门微调 |

## 五、视觉-语言模型微调

| 策略 | 内容 | 适用 |
|------|------|------|
| **仅微调投影层** | 冻结 ViT + LLM，只训练 Projection | 快速适配新模态 |
| **LoRA LLM** | 冻结 ViT，LoRA 微调 LLM | 保持视觉，提升交互 |
| **全量微调** | 全部参数可训练 | 数据量大的情况 |
| **视觉编码器解冻** | 解冻 ViT 最后几层 | 适配特定视觉域 |

---

## 六、微调实践建议

| 场景 | 推荐方法 |
|------|---------|
| 快速实验、单卡 | QLoRA（r=16） |
| 生产部署、追求精度 | LoRA（r=64, 全注意力层） |
| 数据少（<1K） | LoRA + 数据增强 |
| 数据多（>10K） | 考虑全量微调 |
| 领域差异大 | 全量视觉编码器 + LoRA LLM |
