# 如何构建和训练神经网络（上）：训练流程与损失函数

> 西安电子科技大学 · 具身智能微专业
> 课程笔记

## 一、数据集

### 1.1 常见基准数据集

| 数据集 | 内容 | 规模 | 用途 |
|--------|------|------|------|
| MNIST | 手写数字（灰度 28×28） | 70k | 入门分类 |
| CIFAR-10 | 自然图像（彩色 32×32） | 60k | 通用分类 |

### 1.2 数据集的三个子集

| 子集 | 作用 | 使用方式 |
|------|------|---------|
| **训练集**（Train） | 训练模型参数 | 前向传播 + 反向传播 + 更新权重 |
| **验证集**（Val） | 调超参数、监控泛化 | 仅前向传播，不更新权重 |
| **测试集**（Test） | 最终评估 | 只用一次，确认模型真实效果 |

### 1.3 训练流程

```
使用训练集训练模型
    ↓
使用验证集评估模型
    ↓
根据验证集效果调整超参数
    ↓
重复训练，选择验证集效果最好的模型
    ↓
使用测试集确认最终效果
```

> 类比学习过程：**做题 → 批改 → 订正**

## 二、训练循环

一个训练 step 对应四个操作：

```python
outputs = model(inputs)          # 1. 前向传播：输入 → 输出
loss = criterion(outputs, target) # 2. 计算损失：预测 vs 真实值
optimizer.zero_grad()             # 3. 清空上一轮梯度（避免累加）
loss.backward()                   # 4. 反向传播：计算每个参数的梯度
optimizer.step()                  # 5. 参数更新：真正修改网络权重
```

| 步骤 | 方法 | 作用 |
|------|------|------|
| Forward | `model(inputs)` | 输入经过各层计算得到预测值 |
| Loss | `criterion(outputs, target)` | 衡量预测与真实值的差距 |
| Zero Grad | `optimizer.zero_grad()` | 清空上一轮累积的梯度 |
| Backward | `loss.backward()` | 链式法则求导，计算每个参数应如何变化 |
| Step | `optimizer.step()` | 按梯度和学习率更新权重 |

## 三、损失函数

### 3.1 什么是损失函数？

> Loss Function = 模型学习的"评分标准"

```
预测值（Prediction） → 比较 ← 真实值（Ground Truth）
                              ↓
                         误差大小 = Loss
```

损失函数将模型表现量化为一个标量——训练的目标就是最小化这个值。

### 3.2 如何选择损失函数？

核心原则：**损失函数要和任务目标一致**。

| 任务 | 推荐损失函数 | 原因 |
|------|------------|------|
| 回归 | MSE（均方误差） | 惩罚大误差 |
| 回归（有异常值） | MAE（平均绝对误差） | 对异常值更鲁棒 |
| 二分类 | BCE（二元交叉熵） | 概率输出，梯度好 |
| 多分类 | Cross-Entropy（交叉熵） | 与 Softmax 组合，梯度稳定 |

> 为什么 MNIST 使用交叉熵？多分类任务 + 输出层 Softmax，交叉熵的组合梯度形式简洁，收敛快。

### 3.3 训练日志如何解读？

损失下降不代表泛化一定好。三种典型状态：

| 状态 | Train Loss | Val Loss | Accuracy | 含义 |
|------|-----------|---------|----------|------|
| **正常训练** | ↓ 下降 | ↓ 下降 | ↑ 上升 | 模型在学习，泛化良好 |
| **过拟合** | ↓ 继续下降 | ↑ 不降反升 | 变差 | 模型死记硬背训练集 |
| **欠拟合** | 很高 | 很高 | 低 | 模型表达能力不足 |

## 四、过拟合与欠拟合

### 4.1 过拟合（Overfitting）

**原因**：

- 训练数据单一，样本不足
- 训练数据中噪声过大
- 模型过于复杂

**解决方法**：

| 方法 | 说明 |
|------|------|
| 数据增强（Data Augmentation） | 旋转、裁剪、翻转、颜色扰动 |
| 扩大数据集 | 收集更多真实数据 |
| 正则化（L1 / L2） | 约束权重不要过大 |
| Dropout | 随机丢弃神经元，强制冗余学习 |
| 简化模型 | 奥卡姆剃刀原则——简单有效优先 |

### 4.2 欠拟合（Underfitting）

**原因**：特征维度过少，模型过于简单，无法拟合训练数据的规律。

**解决方法**：

- 增加特征维度
- 增加模型复杂度（更多层、更多神经元）
- 精简训练数据与特征（去掉噪声特征）

## 五、优化器

### 5.1 批量梯度下降（Batch Gradient Descent, BGD）

使用全部训练数据计算梯度：

$$w_{new} = w_{old} - \eta \cdot \nabla L_{全部数据}$$

| 维度 | 特点 |
|------|------|
| 梯度方向 | 准确 |
| 计算量 | 大（每步遍历全部数据） |
| 缺陷 | 梯度为零（鞍点/局部极小）时，权重不再更新，梯度下降失效 |

### 5.2 随机梯度下降（Stochastic Gradient Descent, SGD）

每步使用一个或少量样本计算梯度：

$$w_{new} = w_{old} - \eta \cdot \nabla L_{单个或少量样本}$$

| 维度 | 特点 |
|------|------|
| 梯度方向 | 有噪声（波动） |
| 计算量 | 小 |
| 优势 | 噪声帮助跳出鞍点和局部极小 |
| 收敛 | 波动中前进，最终收敛 |

> **偏导数决定爬坡的方向，学习率决定步伐大小。**
