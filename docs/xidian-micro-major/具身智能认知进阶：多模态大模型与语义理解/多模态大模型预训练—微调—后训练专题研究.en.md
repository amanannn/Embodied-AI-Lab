## 🧭 Part 1: Basic Concepts of Multimodal Large Models

### I. Core Evolution of Large Models: PLM vs LLM

1. **PLM (Pre-trained Language Model)**
* **Definition**: A language model pre-trained on large-scale unsupervised corpora.
* **Representative Models**: BERT (bidirectional encoding), GPT-1 / GPT-2 (autoregressive decoding).
* **Characteristics**: Relatively small parameter counts, typically in the **million to billion range**.

2. **LLM (Large Language Model)**
* **Definition**: Refers to language models with massive parameter counts, possessing stronger general-purpose capabilities and emergent abilities.
* **Characteristics**: Extremely large parameter scales, reaching **tens of billions to hundreds of billions+** (e.g., GPT-3 with 175 billion parameters, GPT-4 with approximately 1.8 trillion parameters, PaLM 2 with 340 billion parameters).
* **Core Advantages**: Stronger cross-task and cross-domain generalization capability, more accurate understanding and following of natural language instructions.

### II. Key Core Laws and Effects

1. **Emergence (Emergent Ability)**
* **Definition**: When the model scale reaches a specific critical threshold, it **suddenly exhibits non-linearly improving capabilities that small models do not possess**.
* **Three Typical Manifestations**:
  * **In-context Learning**: The ability to complete entirely new tasks with only a few examples.
  * **Chain-of-Thought Reasoning**: The ability to perform multi-step logical reasoning, progressively decomposing complex problems.
  * **Instruction Following**: Accurately understanding and following complex natural language instructions.
* **Mechanism of Occurrence**: Increased model scale leads to a leap in representational capacity; massive data allows more knowledge and patterns to be captured; complex capabilities are activated at large scales.

2. **Scaling Law**
* **Definition**: The performance (Test Loss) of LLMs **shows predictable and continuous improvement** as model parameter size, training data volume (Tokens), and computational resources increase.
* **Three Elements**: Model Parameter Size (Model Size), Training Data Volume (Data Size), Computational Resources (Compute).
* **Data Bottleneck**: The larger the model scale, the more training data (Token progress) is required. Model parameters and data scale must co-evolve to sustain performance gains.

### III. Five Major Paradigm Shifts (From Single Perception to Super Agent)

| Paradigm Shift Direction | From (Traditional Mode) | To (Large Model Mode) | Core Goal and Value |
| --- | --- | --- | --- |
| **Single Modality $\rightarrow$ Multimodal** | Single text processing / dedicated CV | Deep integration of images, text, audio, video | Break perceptual boundaries, achieve cross-modal comprehensive understanding and reasoning (e.g., GPT-4V). |
| **Prediction $\rightarrow$ Generation** | Discriminative tasks (classification/judgment) | Generative creation (creation/generation) | Usher in the AIGC era, AI possesses the creativity to combine and transform patterns. |
| **Single Task $\rightarrow$ Multi-task** | Train dedicated models for each task | One general-purpose model handles multiple tasks | Unified interface (translation + QA + code, etc.), greatly lowering the barrier to AI applications. |
| **Perception $\rightarrow$ Cognition** | Pure "recognition" layer (vision/speech) | Relationship understanding, logical reasoning, decision-making layer | From "seeing/hearing" to "understanding why seeing/hearing", moving towards general intelligence. |
| **Large Model $\rightarrow$ Agent** | Passive response assistant (Copilot) | Proactive execution agent (AI Agent) | **Agent Architecture**: Set goal $\rightarrow$ Plan steps $\rightarrow$ Call tools $\rightarrow$ Execute actions $\rightarrow$ Complete task. |

### IV. Three Core Directions of Multimodal Foundation Models

1. **Vision Understanding**: Learning general visual representations, from pixels to deep semantic understanding (e.g., GPT-4V, LLaVA, BLIP-2, InstructBLIP, MiniGPT-4). Tasks include image captioning, visual question answering (VQA), image-text retrieval, etc.
2. **Vision Generation**: Learning image/video data distributions to generate high-quality content (e.g., DALL-E 3, Stable Diffusion, Midjourney, Imagen, Sora). Tasks include text-to-image, image-to-image, text-to-video.
3. **Universal Interface**: Unifying multimodal inputs, connecting understanding, generation, and action (e.g., GPT-4 Universal Interface, LangChain orchestration framework, AutoGPT/OpenAI Assistants API agent frameworks).

### V. Representative Technology Deep Dive: BERT vs ViT vs MAE

#### 1. BERT Technical Deep Dive (NLP Milestone)

* **Basic Concept**: A Transformer-based bidirectional pre-trained language model proposed by Google in 2018, inaugurating the **"pre-training + fine-tuning"** new paradigm.
* **Architecture Specifications**:
  * **BERT-Base**: 12-layer Transformer encoder, 110M parameters.
  * **BERT-Large**: 24-layer Transformer encoder, 340M parameters.
* **Input Representation Construction**: $\text{Final Input Embedding} = \text{Token Embedding} + \text{Segment Embedding} + \text{Position Embedding}$
  * **Token Embedding**: Maps words/subwords into dense vectors through a trainable word vector matrix (e.g., dimension 768).
  * **Segment Embedding**: Used to distinguish different sentences in the input (e.g., sentence A is assigned $E_A$, sentence B is assigned $E_B$), primarily serving the Next Sentence Prediction (NSP) task.
  * **Position Embedding**: BERT uses **learnable absolute position encoding** (max sequence length 512, dimension 768), providing the model with order-aware information.
* **Two Core Pre-training Tasks**:
  * **Masked Language Model (MLM)**:
    * **Mechanism**: Similar to cloze testing, randomly selecting **15%** of Tokens in the input sequence for processing.
    * **Dynamic Masking Strategy Distribution**: Of the selected 15%, **80%** are replaced with `[MASK]`, **10%** are replaced with random words (adding noise), **10%** remain unchanged (preserving the original word). This aims to reduce the input inconsistency between pre-training and fine-tuning phases.
    * **Loss**: Cross-entropy loss is computed only for the masked/processed positions ($\mathcal{L}_{MLM}$).
  * **Next Sentence Prediction (NSP)**:
    * **Mechanism**: A binary classification task that uses the output vector at the special `[CLS]` token position to aggregate the relationship information of the entire sentence pair, predicting whether sentence B is the next sentence of sentence A through a fully connected layer and Softmax. The ratio of positive samples (IsNext, consecutive in original text) to negative samples (NotNext, randomly sampled and concatenated) is **1:1**.
* **Downstream Fine-tuning Applications**: Simple output layers can be added on top of the pre-trained model to quickly adapt to tasks such as text classification, question answering systems, named entity recognition (NER), and sentence similarity computation.

#### 2. ViT (Vision Transformer) Technical Deep Dive (The Transformer Era in CV)

* **Core Challenge**: Directly applying Transformers to images results in excessively long sequences (self-attention computational complexity is $O(n^2)$, pixel-level computation is unacceptable).
* **Patching Strategy**: Divide a $224 \times 224$ 2D image into fixed-size patches (e.g., $16 \times 16 \times 3 = 768$ dimensions), serving as "visual words" (similar to Tokens in NLP), forming a 1D sequence (totaling $14 \times 14 = 196$ Tokens), and additionally adding a learnable `[class]` encoding to aggregate global information.
* **Advantages over CNN**: Flexibly handles inputs of different sizes; uses the self-attention mechanism to **directly model long-range, global dependencies**; under large-scale pre-training, has fewer parameters and stronger generalization/transfer capability. However, on small datasets, performance may be slightly inferior due to the lack of CNN's inherent "inductive biases" (locality and translation invariance).
* **Pre-training Task - Masked Patch Prediction (MPP)**: Self-supervised learning. Randomly mask some patches in the image, predict and reconstruct the content of masked patches in pixel space or feature space, minimizing reconstruction error (MSE/L1 loss).

#### 3. MAE (Masked Autoencoders) Technical Deep Dive

* **Asymmetric Architecture**: Proposed by Kaiming He in 2021. Adopts an asymmetric encoder-decoder structure: **heavy encoder (standard ViT), light decoder (few layers, small parameters)**.
* **Extremely High Masking Ratio**: Uses an extremely high masking ratio of **75%** (only 25% of visible patches are fed into the encoder), forcing the model to learn deep global semantics and details of the image from extremely limited information. The decoder processes all patches including mask tokens for pixel-level reconstruction (regression task) during pre-training, and is directly discarded during inference.

#### 📌 Key Differences Between Vision (MAE) and Language (BERT) Pre-training:

* **Information Density**: Text has high information density (rich word semantics, low redundancy), so BERT only requires a low masking ratio of **15%**; images have low information density (extremely high pixel-level redundancy), so MAE requires a high masking ratio of **75%**.
* **Task and Decoder**: BERT performs word-level prediction (classification task), with the task head playing a supporting role; MAE performs pixel-level reconstruction (regression task), with the lightweight decoder being its core reconstruction module.

## ⚙️ Part 2: Multimodal Large Model Pre-training Mechanisms

### I. Evolution of Pre-training Development

* **2013-2015 (Static Word Vector Era)**: Word2Vec, GloVe, FastText. Rely on static vocabularies, based on statistical self-supervised objectives, unable to model contextual changes.
* **2016-2017 (Dynamic Contextual Representation)**: ELMO, CoVe, Seq2Seq. Use bidirectional LSTMs, word representations change dynamically with context.
* **2017 Watershed**: Google Brain publishes the landmark paper **"Attention Is All You Need"**, proposing the **Transformer architecture** based entirely on the attention mechanism. Abandons sequential processing, achieves fully parallel computation, and completely solves the long-range dependency problem.
* **2018-2019 (Transformer Pre-training + Fine-tuning Paradigm)**: BERT (bidirectional encoding / masked modeling), GPT (unidirectional autoregressive generation). Universal representation + transfer learning becomes mainstream.
* **2020-2022 (Large-scale Foundation Model Era)**: GPT-3 (175 billion parameters, inaugurating the in-context learning paradigm), CLIP, DALL-E, MAE.
* **2023+ (Unified Multimodal Era)**: GPT-4V, Gemini, Flamingo, PaLM-E. Multiple modalities such as text, image, speech, and video move towards unification through unified architectures (e.g., MoE / Multimodal Transformer) and "pre-training + instruction fine-tuning + alignment post-training".

### II. Why Do Multimodal Large Models Need Pre-training?

1. **Resolving Representation Space Differences**: The underlying distributions of text (discrete symbols, abstract semantics), images (continuous pixel matrices), and speech (time-frequency signals) differ, making direct alignment impossible. Pre-training can automatically learn a **unified cross-modal semantic embedding space**.
2. **Alleviating Extremely High Annotation Costs**: The cost of manually annotating large-scale multimodal data (e.g., image-text pairs, video-text pairs) grows exponentially. Pre-training leverages massive unlabeled data (e.g., web-crawled data) for self-supervised learning, breaking the annotation bottleneck.
3. **Establishing Cross-modal Alignment and Reuse**: The quality of cross-modal alignment directly determines downstream task performance. Good pre-training alignment can provide a strong parameter initialization starting point for downstream tasks, achieving **"general education" (universal representation) first, then "specialized education" (downstream fine-tuning)**.

### III. Core Value and Process of Pre-training

* **Three Value Manifestations**:
  * **Data Efficiency**: Reduces the dependence of downstream tasks on labeled data by **a factor of 10 - 100**.
  * **Convergence Speed**: Convergence speed during the fine-tuning phase is **improved by 5 - 10 times**.
  * **Performance Leap**: Accuracy/effectiveness on specific downstream tasks improves by an average of **over 20%**.
* **Two-Stage Standard Process**:
  1. **Stage 1: Large-scale Unlabeled/Weakly Labeled Data Pre-training**. Uses internet-scale massive multimodal data, automatically generates supervision signals through self-supervised tasks, producing a foundational base model with general representation capabilities.
  2. **Stage 2: Small-scale Task-specific Data Fine-tuning**. Uses labeled data for specific downstream tasks (e.g., text classification, object detection, QA pairs), continues optimization on top of pre-trained weights, updates parameters to quickly adapt to the application.

### V. Overview of CV Pre-training Task System

1. **Proxy Tasks**: Use artificially defined structures or image attributes to design self-supervised signals from spatial and color relationships. Examples: Jigsaw puzzle, RotNet (rotation angle prediction), Colorization, Inpainting.
2. **Sequence Learning Tasks**: Targeting video data, utilize the temporal coherence of video sequences to learn dynamic representations. Examples: Video Prediction (future frame prediction), Temporal Order Verification, Contrastive Predictive Coding (CPC).
3. **Data Generation and Reconstruction**: Learn features by fitting data distributions or reconstructing corrupted regions.
   * **Generation Tasks**: GAN, VAE, PixelCNN, Diffusion Models, etc.
   * **Reconstruction Tasks**: Masked Image Modeling (MIM, e.g., MAE, BEIT, SimMIM), etc.
4. **Contrastive Learning**: Learn strongly discriminative semantic features by maximizing the consistency of different augmented views of the same image (positive samples) and minimizing the similarity of different images (negative samples) (e.g., SimCLR, MoCo, BYOL, SWAV, DINO).

### 🚀 Extended Supplement: Frontier Architectures and Technological Evolution

#### 1. The Leap of the GPT Series (From "Fine-tuning" to "Prompting" and "In-context Learning")

* **GPT-1 (2018)**: Combines unsupervised autoregressive pre-training (objective: predict the next word $P(w_t \vert w_{<t})$) with supervised fine-tuning. Uses **masked multi-head self-attention mechanism**, ensuring positions can only attend to historical information at positions $\le i$.
* **GPT-2 (2019)**: Proposes unified unsupervised learning, **cancels fine-tuning for specific downstream tasks**, relying entirely on multi-task shared parameter space.
* **GPT-3 (2020)**: Parameter scale leaps to 175 billion. Core innovation introduces **In-Context Learning**, requiring no gradient updates or fine-tuning; only a Prompt with a few examples (Few-shot / Zero-shot) enables the large model to perform diverse tasks.
* **GPT-4 $\rightarrow$ GPT-5 Evolution Prediction**:
  * **GPT-4 (2023)**: Multimodal Transformer, natively supports text + image input, uses RLHF to reinforce value alignment and safety.
  * **GPT-5 (Expected)**: Moving towards **multimodal MoE (Mixture of Experts) sparse architecture**, with longer context, stronger deep thinking and mathematical/scientific reasoning, and advanced super-agent (Agent) autonomous planning and tool-use capabilities.

#### 2. Sparse Mixture of Experts Architecture (MoE)

* **Core Mechanism**: Composed of a gating network (Router) and multiple parallel expert networks (FFN). When each Token is input, the gating network computes weights via Softmax, **sparsely activates** the most suitable Top-k experts (typically $k=2$) for computation, and finally produces a weighted sum output ($y = \sum g_i(x)E_i(x)$).
* **Core Advantage**: Achieves **10x+** expansion of model parameter capacity without increasing the computational cost and latency of a single inference.

#### 3. DeepSeek Efficiency Revolution Innovations

* **DeepSeekMoE**: Extreme expert specialization with sparse activation, maintaining extremely low computation and hardware costs at trillion-parameter scale.
* **MLA Attention Mechanism (Multi-head Latent Attention)**: By performing **low-dimensional joint compression (Latent Vector)** on Key (K) and Value (V), greatly reduces the memory usage of KV Cache during long-sequence inference, completely breaking the hardware bottleneck for large-model long-text reasoning.
* **Sparse Attention**: Reduces the standard Attention's $O(L^2)$ quadratic complexity to $O(L \cdot k)$ linear complexity.

### 📝 Extended Thought Question: How to Combine BERT and ViT to Design a Multimodal Model?

**Reference the classic paradigm design of CLIP (Contrastive Language-Image Pre-training):**

1. **Text Encoder (BERT-based)**: Bidirectionally encode the text sequence to extract sentence-level and word-level multi-dimensional semantic vectors.
2. **Visual Encoder (ViT-based)**: Divide the image into a sequence of patches and feed into a Transformer to extract global visual feature vectors.
3. **Cross-modal Contrastive Learning Alignment**: Project text and image features into the **same multimodal shared representation space** through a unified mapping layer. During training, maximize the feature dot product distance for matching image-text pairs (positive samples) and minimize the similarity for non-matching pairs (negative samples).
4. **Application Potential**: After constructing a unified multimodal shared space, it can be perfectly transferred downstream to **cross-modal retrieval (image search for text / text search for image)**, **image-text matching judgment**, and **zero-shot generalization classification** among other diverse multimodal understanding tasks.
