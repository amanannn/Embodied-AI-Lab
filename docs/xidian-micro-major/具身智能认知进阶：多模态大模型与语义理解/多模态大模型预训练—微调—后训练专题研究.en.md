## I. Basic Concepts of Multimodal Large Models

### 1.1 Core Evolution: PLM vs LLM

| Dimension | PLM (Pre-trained Language Model) | LLM (Large Language Model) |
|-----------|--------------------------------|----------------------------|
| **Parameter Scale** | Millions to billions | Billions to trillions+ |
| **Representative Models** | BERT (bidirectional), GPT-1/2 (autoregressive) | GPT-3 (175B), GPT-4 (~1.8T), PaLM 2 (340B) |
| **Core Capability** | Transfer learning, fine-tune for downstream tasks | Cross-task generalization, instruction following, emergence |

### 1.2 Key Laws and Effects

#### Emergence

When model scale reaches a specific critical threshold, capabilities **suddenly emerge that small models do not possess — non-linear leaps in performance**.

| Three Classic Manifestations | Description |
|----------------------------|-------------|
| **In-context Learning** | Complete entirely new tasks with only a few examples |
| **Chain-of-Thought** | Multi-step logical reasoning, decomposing complex problems |
| **Instruction Following** | Accurately understand and follow complex natural language instructions |

> Mechanism: scale expansion → representational capacity leaps; massive data captures more knowledge and patterns; complex abilities are activated at scale.

#### Scaling Law

LLM performance (Test Loss) shows **predictable continuous improvement** as model size, training data volume, and compute increase.

| Three Factors | Description |
|--------------|-------------|
| Model Size | Larger → stronger, but must match data |
| Data Size (Tokens) | Larger models need more data |
| Compute | GPU-hours grow quadratically |

> Data bottleneck: model parameters and data scale must **scale together** for sustained performance improvement.

### 1.3 Five Paradigm Shifts

| Shift | From (Traditional) | To (Large Model Era) | Core Goal |
|-------|-------------------|---------------------|-----------|
| **Unimodal → Multimodal** | Single text / dedicated CV | Deep fusion of image, text, audio, video | Cross-modal understanding and reasoning |
| **Prediction → Generation** | Discriminative (classification) | Generative creation | Usher in the AIGC era |
| **Single-task → Multi-task** | One model per task | One universal model for many tasks | Unified interface, lower barriers |
| **Perception → Cognition** | Recognition layer | Relational understanding, logical reasoning | From "seeing" to "understanding why" |
| **LLM → Agent** | Passive assistant (Copilot) | Proactive agent (AI Agent) | Goal → Plan → Tools → Execute → Complete |

### 1.4 Three Core Directions of Multimodal Foundation Models

| Direction | Positioning | Representative Models | Typical Tasks |
|-----------|------------|----------------------|---------------|
| **Vision Understanding** | From pixels to deep semantic understanding | GPT-4V, LLaVA, BLIP-2, MiniGPT-4 | Image captioning, VQA, cross-modal retrieval |
| **Vision Generation** | Learn data distribution, generate high-quality content | DALL-E 3, Stable Diffusion, Midjourney, Sora | Text-to-image, image-to-image, text-to-video |
| **Universal Interface** | Unify multimodal I/O, connect understanding/generation/action | GPT-4 API, LangChain, AutoGPT | Agent orchestration and tool use |

## II. Representative Technical Deep Dives: BERT vs ViT vs MAE

### 2.1 BERT — NLP Pre-training Landmark

Proposed by Google in 2018. Bidirectional Transformer encoder; pioneered the **"pre-train + fine-tune"** paradigm.

| Specification | Layers | Parameters |
|--------------|--------|------------|
| BERT-Base | 12 Transformer encoder layers | 110M |
| BERT-Large | 24 Transformer encoder layers | 340M |

#### Input Representation

$$\text{Input Embedding} = \text{Token Embedding} + \text{Segment Embedding} + \text{Position Embedding}$$

| Embedding Type | Description |
|---------------|-------------|
| **Token Embedding** | Words/subwords mapped to dense vectors via trainable embedding matrix (dim 768) |
| **Segment Embedding** | Distinguishes sentences ($E_A$ / $E_B$), serves NSP task |
| **Position Embedding** | Learnable absolute position encoding (max length 512) |

#### Two Core Pre-training Tasks

| Task | Mechanism | Loss |
|------|-----------|------|
| **MLM (Masked Language Model)** | Randomly select 15% tokens: 80%→`[MASK]`, 10%→random word, 10%→keep original | Cross-entropy only on masked positions |
| **NSP (Next Sentence Prediction)** | `[CLS]` aggregates sentence-pair info → binary classification (1:1 positive/negative) | Sentence-level cross-entropy |

> The 80-10-10 strategy for MLM bridges the gap between pre-training and fine-tuning — downstream data has no `[MASK]` tokens.

### 2.2 ViT — CV Enters the Transformer Era

#### Core Challenge and Solution

Applying Transformer directly to images causes prohibitively long sequences: $O(n^2)$ self-attention at pixel level is infeasible.

**Patching Strategy**: Slice $224 \times 224$ image into $16 \times 16$ patches, each flattened to a 768-dim vector, forming 196 "visual tokens" + one learnable `[class]` token.

| Dimension | CNN | ViT |
|-----------|-----|-----|
| Receptive Field | Local → expands layer by layer | Global self-attention from the start |
| Inductive Bias | Built-in locality + translation invariance | None — requires massive data |
| Small Dataset | Better | May underperform CNN |
| Large Dataset | Saturates | Continues improving, surpasses CNN |

#### Pre-training Task — Masked Patch Prediction (MPP)

Randomly mask some patches, predict and reconstruct masked content in pixel or feature space (MSE/L1 loss).

### 2.3 MAE — Masked Autoencoders

Proposed by Kaiming He in 2021. Asymmetric architecture: **heavy encoder (standard ViT), lightweight decoder** (fewer layers, fewer parameters).

| Feature | Description |
|---------|-------------|
| **Extreme Mask Ratio** | **75%** — only 25% visible patches fed to encoder |
| **Design Intent** | Force the model to learn deep global semantics from extremely limited information |
| **Decoder** | Processes all patches (including mask tokens) during pre-training for pixel-level reconstruction; discarded at inference |

#### Vision (MAE) vs Language (BERT) Pre-training Differences

| Dimension | BERT (Language) | MAE (Vision) |
|-----------|----------------|--------------|
| **Mask Ratio** | 15% (low) | 75% (extreme) |
| **Information Density** | Text is semantically rich, low redundancy | Images have extremely high pixel-level redundancy |
| **Task Type** | Token-level prediction (classification) | Pixel-level reconstruction (regression) |
| **Decoder** | Task head plays auxiliary role | Lightweight decoder is the reconstruction core |

## III. Pre-training Mechanisms

### 3.1 Evolution of Pre-training

| Period | Stage | Representative Technology | Characteristic |
|--------|-------|--------------------------|----------------|
| 2013-2015 | Static Word Vectors | Word2Vec, GloVe, FastText | Statistics-based, no context modeling |
| 2016-2017 | Dynamic Context | ELMO, CoVe, Seq2Seq | Bidirectional LSTM, context-dependent representations |
| **2017** | **Watershed** | **Transformer** | Attention Is All You Need — parallelization + long-range dependency |
| 2018-2019 | Pre-train + Fine-tune | BERT, GPT | Universal representations + transfer learning become mainstream |
| 2020-2022 | Large-scale Foundation Models | GPT-3, CLIP, DALL-E, MAE | Hundred-billion parameters + in-context learning |
| 2023+ | Unified Multimodal | GPT-4V, Gemini, PaLM-E | Multiple modalities unified via one architecture |

### 3.2 Why Pre-training?

| Reason | Description |
|--------|-------------|
| **Resolve Representation Space Gaps** | Text (discrete symbols), images (continuous pixels), and speech have different underlying distributions; pre-training learns a unified cross-modal semantic embedding space |
| **Alleviate Extreme Annotation Costs** | Multimodal data annotation costs grow exponentially; pre-training uses massive unlabeled data for self-supervised learning |
| **Establish Cross-modal Alignment** | "General education" (universal representations) first, then "specialized education" (downstream fine-tuning) |

### 3.3 Core Value of Pre-training

| Value | Quantified Effect |
|-------|-------------------|
| **Data Efficiency** | Downstream labeled data needs reduced by 10–100× |
| **Convergence Speed** | Fine-tuning convergence 5–10× faster |
| **Performance Leap** | Downstream accuracy improved by 20%+ on average |

**Two-stage Standard Pipeline**:

```
Stage 1: Large-scale unlabeled/weakly-labeled pre-training
    → Self-supervised tasks auto-generate supervisory signals
    → Produce a base model with universal representation capability

Stage 2: Small-scale task-specific fine-tuning
    → Continue optimizing on top of pre-trained weights
    → Quickly adapt parameters for specific applications
```

## IV. NLP Core Pre-training Tasks

| Task | Mechanism | Training Signal | Input Example | Core Capability | Representative Models |
|------|-----------|----------------|---------------|-----------------|----------------------|
| **MLM** | Mask ~15% tokens, predict from bidirectional context | Token-level multi-class CE | `I went to the [MASK] to study` | Bidirectional context modeling | BERT, RoBERTa, ALBERT |
| **DAE** | Add noise (delete/shuffle/replace), train to denoise | Sequence reconstruction CE | Noisy input → clean output | Robust feature representation | BART, T5, SpanBERT |
| **RTD** | Small generator replaces some words, large discriminator judges each word real/fake | Binary CE | Labels: `[real,real,real,fake,real,fake]` | 100% sample utilization, fine-grained semantics | ELECTRA, DeBERTa |
| **NSP** | Judge whether B logically follows A | Sentence-level binary CE | `[CLS] A [SEP] B [SEP]` | Inter-sentence coherence (somewhat easy) | BERT (original) |
| **SOP** | Challenging NSP replacement — judge whether two segments are in correct order | Sentence-level binary CE | Positive: A→B / Negative: B→A | Deep logical understanding between sentences | ALBERT |

> RTD replaces generation with discrimination for 100% sample utilization. SOP avoids topic leakage, forcing models to understand causality and logical transitions.

## V. CV Pre-training Task Taxonomy

### 5.1 Four Categories

| Category | Mechanism | Representative Methods |
|----------|-----------|----------------------|
| **Proxy Tasks** | Use image structure to design self-supervised signals | Jigsaw, RotNet, Colorization, Inpainting |
| **Sequence Learning** | Leverage temporal coherence in video | Future frame prediction, Temporal Order Verification, CPC |
| **Generation & Reconstruction** | Fit data distribution or reconstruct corrupted regions | GAN, VAE, Diffusion Models / MIM (MAE, BEIT, SimMIM) |
| **Contrastive Learning** | Pull positive pairs together, push negative pairs apart | SimCLR, MoCo, BYOL, SWAV, DINO |

## VI. Frontier Architectures and Technical Evolution

### 6.1 GPT Series: From Fine-tuning to In-context Learning

| Version | Year | Core Contribution |
|---------|------|-------------------|
| **GPT-1** | 2018 | Unsupervised autoregressive pre-training + supervised fine-tuning; masked multi-head self-attention |
| **GPT-2** | 2019 | Eliminated task-specific fine-tuning, unified unsupervised multi-tasking |
| **GPT-3** | 2020 | 175B parameters; in-context learning — no gradient updates, Prompt + Few-shot enables diverse tasks |
| **GPT-4** | 2023 | Multimodal Transformer, native text+image, RLHF value alignment |
| **GPT-5 (expected)** | — | Multimodal MoE sparse architecture, longer context, advanced Agent autonomous planning |

### 6.2 Mixture of Experts (MoE)

| Component | Function |
|-----------|----------|
| **Router (Gating Network)** | Computes Softmax weights for each token |
| **Sparse Activation** | Only Top-k (typically k=2) experts activated per token |
| **Output** | Weighted sum $y = \sum g_i(x)E_i(x)$ |

> Achieves 10×+ parameter capacity expansion without increasing per-inference computation cost.

### 6.3 DeepSeek Efficiency Revolution

| Innovation | Mechanism | Effect |
|-----------|-----------|--------|
| **DeepSeekMoE** | Extreme expert specialization with sparse activation | Trillion-parameter scale at very low compute cost |
| **MLA (Multi-head Latent Attention)** | Low-dimensional joint compression of K and V (Latent Vector) | Drastically reduces KV Cache memory, breaks long-text inference hardware bottleneck |
| **Sparse Attention** | $O(L^2)$ → $O(L \cdot k)$ linear complexity | Efficient long-sequence processing |

## VII. Thought Exercise: Designing a Multimodal Model with BERT + ViT

Reference: **CLIP (Contrastive Language-Image Pre-training)** paradigm:

| Step | Component | Function |
|------|-----------|----------|
| 1 | **Text Encoder (BERT-based)** | Bidirectionally encode text, extract sentence/word-level semantic vectors |
| 2 | **Vision Encoder (ViT-based)** | Patch images → Transformer, extract global visual features |
| 3 | **Cross-modal Contrastive Alignment** | Unified projection layer → project text and image features into shared space |
| Training | Positive pairs close / negative pairs far | Maximize similarity of matched image-text pairs |
| Application | Cross-modal retrieval / Zero-shot classification | Text→Image search, Image→Text search, zero-shot generalization |

> CLIP's core insight is not architectural novelty but proof that **400M image-text pairs** + a simple enough objective (contrastive learning) can learn universal cross-modal representations.
