## I. From the Yanshi Automaton to Modern Robots

### 1.1 A Brief History of Robotics

> "King Mu stared in wonder — the figure walked, bowed, and looked like a real person. Truly a craftsman! Touch its chin and it would sing in tune; raise its hands and it would dance in rhythm." — *Liezi · Tang Wen*

| Period | Milestone | Significance |
|--------|-----------|-------------|
| 4th century BC | Archytas' steam-powered "pigeon" flyer | Earliest mechanical self-propelled device |
| 1495 | Da Vinci's humanoid robot sketch | Could sit up, wave arms, move head |
| 1961 | Unimate industrial robot | Robots transformed from "toys" to "tools" |
| 1973 | KUKA FAMULUS (six-axis) | Programmable autonomous operation |
| 2000 | Honda ASIMO | Bipedal running, stair climbing |
| 2013 | Boston Dynamics Atlas | Extraordinary motion control |

### 1.2 Beyond Industrial Robots

Industrial robots are relatively mature. People began exploring robots in more scenarios:

| Scenario | Representative | Key Technologies |
|----------|---------------|------------------|
| Medical | Minimally invasive surgical robots | Vision + precision control |
| Logistics | Warehouse transport robots | Vision + SLAM |
| Service | Exhibition guide robots | Vision + Speech + NLP |
| Home | Robot vacuums | Vision + SLAM |
| Humanoid | Intelligent humanoid robots | Full technology stack |

> The new focus is no longer on motion capability, but on **intelligence**.

## II. What is an Intelligent Robot?

### 2.1 Two Core Dimensions

Robot → Intelligent Robot ≈ Human, requiring:

| Dimension | Definition | Current Status |
|-----------|-----------|----------------|
| **Autonomy** | Minimal human intervention | Achieved in some scenarios |
| **Generalization** | Adapting to new scenes and tasks | Still a huge challenge |

### 2.2 AI Evolution

| Period | Stage | Capability |
|--------|-------|------------|
| 1956–1960s | Symbolic reasoning | Theorem proving |
| 1970–1980s | Expert systems | Medical/chemical domains |
| 1990s–2010 | Machine learning | Real-world problems |
| 2011–2022 | Deep learning | Images/text/speech |
| **2022–present** | **Large models** | **General-purpose tasks** |

> The strongest LLM + the most advanced humanoid robot = Intelligent robot? The answer lies in "Embodied Intelligence."

## III. How to Build an Intelligent Robot?

### 3.1 Hardware Layer

| Sensor | Function |
|--------|----------|
| RGB Camera | 2D visual signals |
| 3D Camera / LiDAR | 3D point cloud signals |
| Tactile / Force feedback | Contact and force sensing |
| Pose sensors | Proprioception |
| Microphone | Speech signals |

> We can already build robot hardware with basic performance and high-precision sensors.

### 3.2 Software & Algorithm Layer

| Layer | Role | Function | Input → Output |
|-------|------|----------|---------------|
| **Brain** | Embodied Perception + Embodied Reasoning | Comprehensively analyze environment + self-state, make decisions and plans | Sensor data → Decisions/Plans |
| **Cerebellum** | Embodied Execution | Convert plans into motion commands | Plans → Motion commands |
| **Actuators** | Motion control execution | Execute motion commands | Commands → Physical actions |

```
Sensors → Brain (Perception + Reasoning) → Cerebellum (Execution) → Actuators (Motion Control)
   ↑__________________________________________________________________________↓
                                      Closed-loop feedback
```

### 3.3 Example: Cleaning Up Spilled Coffee

Task decomposition:
1. Right the cup and remove the lid
2. Find the cloth
3. Wipe the floor with the cloth
4. Put the cloth back
5. Discard the cup and lid

The robot needs to: perceive (cup/coffee/cloth) → reason through steps → generate trajectories → execute actions.

### 3.4 Maturity of the Three Core Components

| Component | Maturity | Notes |
|-----------|----------|-------|
| **Embodied Perception** | ⭐⭐⭐ | Multimodal LLMs like LLaVA can already perform multi-sensor analysis |
| **Embodied Reasoning** | ⭐⭐ | Mainstream LLMs underperform on embodied reasoning benchmarks |
| **Embodied Execution** | ⭐ | The biggest bottleneck |

#### Two Approaches to Execution

| Approach | Success Rate | Smoothness | Generalization |
|----------|-------------|------------|----------------|
| Multimodal LLM-generated commands | Lower (60%–70%) | Not smooth | Object generalization |
| Diffusion small models | Higher (90%+) | Smooth | Position generalization or none |

> Generating skill library APIs or code APIs: real-world scenarios are too complex — building a complete skill library is nearly impossible. Therefore, current AI is not yet sufficient to make robots truly intelligent — **Embodied Intelligence is needed**.

## IV. What is Embodied Intelligence?

### 4.1 The Key Experiment: Held & Hein, 1963

| Group | Condition | Result |
|-------|-----------|--------|
| **Active** | Could move freely; objects in view changed with movement | **Learned to walk**; realized "objects getting bigger = approaching" |
| **Passive** | Was moved; only saw objects getting bigger/smaller | Did NOT learn to walk |

> Having the condition to walk enables learning to walk — a physical body enables interaction. This is the essential meaning of "embodiment."

### 4.2 Definition

| Level | Definition |
|-------|-----------|
| **Definition 1** | An intelligent system based on a physical body for perception and action, acquiring information, understanding problems, making decisions, and realizing actions through agent-environment interaction |
| **Essence** | A research paradigm emphasizing that intelligent agents with physical bodies gain intelligence through interaction with the physical environment |

> Abstract intelligence (Go, text processing) vs. Embodied intelligence (recognizing objects after moving the occlusion away) — embodied intelligence requires environmental interaction to emerge.

### 4.3 Embodied AI System Architecture

```
Environment (Real ↔ Simulation ↔ Sim2Real)
    ↕
Agent
    ├── Perception (Objects / Scenes / Humans)
    ├── Reasoning (Task planning: Rule-based / LLM-based)
    └── Execution (Navigation / Embodied QA / Interaction)
```

## V. Embodied Perception (具身感知)

### 5.1 Four Categories of Perception

| Type | Content | Sensors | Downstream Tasks |
|------|---------|---------|-----------------|
| **Object Perception** | Geometry, articulation, physical properties | RGB + 3D camera | Pose estimation, grasping, affordance prediction |
| **Scene Perception** | Scene reconstruction & understanding | RGB + Depth | Active exploration, spatial relations, temporal change detection |
| **Behavior Perception** | Gestures, pose, human behavior understanding | RGB + Tactile | Social navigation, autonomous driving |
| **Expression Perception** | Emotion, intention | Vision + Microphone | Human-robot collaboration |

> Robot types that especially need perception capabilities: service robots, collaborative robots, social navigation robots, environment exploration robots.

### 5.2 Perception Pipeline

```
Sensors (RGB / RGB-D / Microphone / Tactile / Thermal)
    → Information encoding (Representation / Modeling / Reconstruction)
        → Information understanding
            → Downstream tasks
```

### 5.3 3D Data Representation & Encoding

| Data Format | Description | Source | Encoding Method |
|-------------|-------------|--------|-----------------|
| **Point Cloud** | Set of (x,y,z) points + features | LiDAR | PointNet, PointNet++ |
| **Mesh** | Points, edges, faces representing object surfaces | CAD, point cloud conversion | MeshNet |
| **Voxel** | Cubic grid | Point cloud conversion | VoxelNet, DeepSDF, Occupancy Network |
| **Depth Map** | Depth value per pixel | Stereo / Structured Light / ToF | GVCNN |

### 5.4 3D Encoding Methods in Detail

| Method | Core Idea | Paper |
|--------|----------|-------|
| **PointNet** | MLP encodes each point → aggregates into global features | Qi et al., CVPR 2017 |
| **PointNet++** | Hierarchical sampling + local feature capture (PointNet improvement) | Qi et al., NIPS 2017 |
| **MeshNet** | MLP+CNN encodes spatial and structural features of each face | Feng et al., AAAI 2019 |
| **VoxelNet** | Voxelization + MLP+CNN encoding | Zhou et al., CVPR 2018 |
| **DeepSDF** | Neural network fits continuous SDF function: voxel coordinates → distance to surface | Park et al., CVPR 2019 |
| **Occupancy Network** | Neural network fits voxel occurrence probability function | Mescheder et al., CVPR 2019 |

> PointNet, MeshNet, and VoxelNet treat 3D data as **discrete units** for convolutional encoding (similar to 2D image encoding in CV). DeepSDF and Occupancy Network use **continuous functions** to represent object shapes — not limited to a single object by introducing a Code as a shape label.

### 5.5 Downstream: Object Pose Estimation

Pose estimation: predicting the 6-DoF pose of an object in 3D space (3 translation + 3 rotation).

| Type | CAD Model Required? | Method |
|------|---------------------|--------|
| **Instance-level** | Yes | Known center point and initial orientation → estimate current pose |
| **Category-level** | No | "Preset" a center point and orientation based on the average of similar objects in the training set |

> NOCS (Normalized Object Coordinate Space): predict standard-space coordinates (x,y,z) at each pixel; combine with depth to obtain translation and orientation.

### 5.6 Perception Research and Embodied AI

> Just as syntactic parsing and part-of-speech tagging were to early NLP, object perception is the "intermediate task" of embodied AI — supporting downstream reasoning and execution while meeting practical application needs.

- Some argue that an **explicit world model** is the next direction for AI, in which perception takes on even greater significance
- 3D data processing methods are highly similar to 2D image processing — large model methodologies can be migrated to 3D

## VI. Current Bottlenecks and Outlook

### 6.1 Status of the Three Components

| Component | Status | Key Challenge |
|-----------|--------|---------------|
| Embodied Perception | Relatively mature | Real-time performance and robustness in complex scenes |
| Embodied Reasoning | Developing | Low LLM success rate, limited generalization |
| Embodied Execution | Biggest bottleneck | Incomplete skill library, insufficient end-to-end trajectory quality |

### 6.2 Core Tensions

1. **Scarce training data**: Robot data is far less than text/image data — Open X-Embodiment attempts to mitigate this through open platforms
2. **Cross-embodiment generalization**: Different robot morphologies require different action spaces
3. **Sim2Real Gap**: Policies trained in simulation struggle to transfer to the real world
4. **Real-time requirements**: The perception → reasoning → execution loop requires low latency

## VII. Summary

```text
Embodied AI = Physical Body + Interaction + Learning

                      ┌── Object Perception (geometry/articulation/physics)
                      ├── Scene Perception (reconstruction/understanding)
          ┌── Perception ─┤
          │               ├── Behavior Perception (gestures/pose/behavior)
          │               └── Expression Perception (emotion/intention)
          │
Embodied ─┼── Reasoning ── Task Planning (rule-based / LLM-based)
  AI      │
          └── Execution ── Navigation / Manipulation / Interaction
                           (APIs / Joint angles / Code)
```

> Embodied AI in the era of large models is not simply about putting GPT into a robot. Perception is relatively mature, reasoning is catching up, and execution is the biggest bottleneck. 3D large models may be the next breakthrough.
