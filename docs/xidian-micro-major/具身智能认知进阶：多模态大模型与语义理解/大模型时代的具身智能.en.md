## 1. From Robot to Intelligent Robot

### 1.1 What is an Intelligent Robot?

From the legendary artisan Yanshi who built a humanoid automaton in ancient China to the present day, robots have evolved from mechanical bodies to intelligent agents. The key differences between intelligent robots and traditional robots lie in two dimensions:

| Dimension | Traditional Robot | Intelligent Robot |
|-----------|-------------------|-------------------|
| **Autonomy** | Pre-programmed, fixed workflow | Perceive environment → Autonomous decision → Execution |
| **Generalization** | Single scenario, fails when environment changes | Adapts to new scenes and new tasks |

> Has AI truly made robots intelligent? Hardware, sensors, and motion control technologies are relatively mature, but at the software algorithm level -- embodied perception, reasoning, and execution -- enormous challenges remain.

### 1.2 Software and Hardware Architecture for Building Intelligent Robots

| Layer | Function | Input | Output |
|-------|----------|-------|--------|
| **Hardware Layer** | Perception and execution foundation | 2D vision / 3D point cloud / tactile / force feedback / pose / voice | --- |
| **Brain** | Embodied perception + embodied reasoning | Multi-sensor information + self-state | Decision and motion planning |
| **Cerebellum** | Embodied execution | Motion commands | Lower-level motion control (code / skill library API / joint angles) |

```
Hardware Perception → Brain (Perception + Reasoning) → Cerebellum (Execution) → Lower-level Motion Control
    ↑____________________________________________________________________↓
                            Closed-loop Feedback
```

## 2. Evolution of Vision Foundation Models: From Pixels to Semantics

### 2.1 Early Vision: What the Machine First Sees Are Pixels

Early computer vision started with "computable" rather than "understandable" -- cameras sample the continuous world into discrete pixel matrices, and algorithms process grayscale values, colors, and gradients.

| Stage | Method | What It Solves |
|-------|--------|----------------|
| **Image Processing** | Filtering, enhancement, denoising | Reduce noise, highlight structures |
| **Edge Detection** | Canny, Sobel | Find contours from grayscale intensity changes |
| **Corner Detection** | Harris | Establish local correspondences between images |
| **Handcrafted Features** | SIFT, HOG, ORB | Humans design rules, machines execute extraction |

> The ceiling of traditional vision: Can match, but struggles to understand. Changes in lighting, pose, and occlusion all break handcrafted rules.

### 2.2 Statistical Learning Era: Feature Extraction + Classifier

| Stage | Role | Typical Method |
|-------|------|----------------|
| Preprocessing | Normalization, grayscale conversion | Pixel operations |
| Feature Extraction | Image → comparable vector | SIFT / HOG / LBP |
| Classification | Output class based on feature vector | SVM / Boosting / Random Forest |
| Post-processing | Refine boundaries, filter noise | NMS / Threshold filtering |

> In this stage, system performance depended on feature quality -- features were handcrafted, and recognition ceilings were locked in by human-designed rules.

### 2.3 Deep Learning Era: Features Begin to Be Learned by the Model Itself

The core breakthrough of CNNs: **Features are no longer written in rules, but stored in model parameters.**

#### Hierarchical Representation in CNNs

| Layer | Extracted Content | Role |
|-------|------------------|------|
| **Low-level** | Edges, colors, textures | Local visual cues |
| **Mid-level** | Local shapes, parts, repeating patterns | Composition into structures |
| **High-level** | Object categories, scene concepts | Semantic understanding |

#### Milestones

| Model | Year | Core Contribution |
|-------|------|-------------------|
| CNN (LeNet / AlexNet) | 1998 / 2012 | End-to-end trainable feature extraction |
| VGG | 2014 | Deep networks + small convolutional kernel standardization |
| **ResNet** | 2015 | Residual connections -- enabling stable training of deep networks |
| Backbone + Head | --- | Generic backbone + task-specific head, the prototype of feature reuse |

### 2.4 Expansion of Vision Tasks: From Classification to Structured Output

| Task | Output Granularity | Question Answered | Representative Method |
|------|--------------------|-------------------|-----------------------|
| **Classification** | Image-level | "What is in the image?" | ResNet, ViT |
| **Detection** | Region-level | "Where is the object?" | R-CNN, YOLO, DETR |
| **Segmentation** | Pixel-level | "Where is the boundary?" | FCN, Mask R-CNN, SAM |

> For robots, a mask is closer to a graspable, projectable operation region than a bounding box.

### 2.5 Limitations of Task-specific Models: Strong Capability but Weak Generalization

Vision models in the CNN era were powerful, but most remained **task-specific**:

| Limitation | Manifestation |
|------------|---------------|
| Closed categories | Can only recognize categories defined in the training set |
| Retrain for new tasks | Classification / detection / segmentation require different heads |
| Weak language interaction | Cannot directly accept natural language input |

## 3. Vision Foundation Models: A Universal Vision Backbone

### 3.1 A New Paradigm: Pretraining → Multi-task Transfer

Vision models shifted from "training for a specific task" toward "first train a general-purpose model, then transfer to multiple tasks."

### 3.2 The Inspiration of Transformer

The 2017 paper "Attention Is All You Need" provided a cross-modal reusable modeling language: **Token + Attention**.

### 3.3 ViT: Images Can Also Be Cut into Tokens

| Step | Operation |
|------|-----------|
| 1. Patch | Divide the image into fixed-size patches |
| 2. Visual Token | Map each patch into a visual token |
| 3. Self-Attention | Transformer models the global relationships among all tokens |

> ViT brought vision into an architectural paradigm similar to language models, laying the foundation for unified multimodal modeling.

### 3.4 Self-Supervised Learning: Breaking Free from Annotation Dependence

| Method | Core Idea |
|--------|-----------|
| **Contrastive Learning** | Bring similar images closer, push dissimilar images apart |
| **Masked Reconstruction (MAE)** | Mask parts, infer the whole from local context |
| **Self-Distillation (DINO)** | Teacher-student mechanism for learning stable representations |

## 4. Open-Vocabulary Perception: Enabling Vision to Understand Natural Language

### 4.1 From Closed Categories to Open Vocabulary

Traditional models rely on a fixed category table. In real-world robot tasks, users do not command in terms of dataset category labels, but rather in natural language:

> "Pick up the red cup on the left side of the table"

| Traditional | Open Vocabulary |
|-------------|-----------------|
| Category label: "cup" | Category + attribute + location + task |
| Closed set | Natural language query |

### 4.2 CLIP: Bringing Images and Language into the Same Space

CLIP's core innovation: **Dual image-text towers + contrastive learning**.

| Component | Function |
|-----------|----------|
| Image Encoder | Image → visual vector |
| Text Encoder | Prompt → language vector |
| Shared Semantic Space | Correct image-text pairs are pulled together, incorrect pairs are pushed apart |

#### CLIP's Capabilities and Boundaries

| ✅ Can Do | ❌ Cannot Directly Do |
|-----------|-----------------------|
| Image-text retrieval | Precise object bounding boxes |
| Zero-shot classification | Pixel-level masks |
| Semantic matching | Robot-operable boundaries |

> CLIP enables vision to understand language, but it does not yet tell the robot precisely where a target is. This is insufficient for embodied intelligence.

### 4.3 Grounding DINO: The Bridge from Text to Region Bounding Boxes

CLIP outputs whole-image semantic scores; Grounding DINO goes further to output **2D object bounding boxes related to the text**.

| Component | Role |
|-----------|------|
| Image Backbone | Extract multi-scale visual features |
| Text Encoder | Encode the prompt into text tokens |
| **Feature Enhancer** | Bidirectional attention interaction between text tokens and image regions |
| Decoder | Output object boxes matching the text semantics |

```
Input: Image + "red cup on the left"
  → Feature Enhancer (text-constrained injection into visual regions)
    → Decoder → Output candidate boxes + text matching scores
```

> The significance of Grounding DINO: **The "object in language" acquires image coordinates for the first time.**

### 4.4 SAM: From Boxes to Pixel-Level Segmentation

SAM (Segment Anything Model) provides general-purpose segmentation capabilities. Used together with Grounding DINO:

```
Natural Language → Grounding DINO → Candidate Boxes → SAM → Pixel-level Mask → Robot-operable Region
```

This forms the complete technical pipeline for open-vocabulary perception.

## 5. The Embodied Intelligence Task Chain

A robot completing a task requires forming a **closed loop from perception to execution**:

| Stage | Function | Input | Output |
|-------|----------|-------|--------|
| **01 Perception** | Acquire environment state | Camera / depth / tactile | Image / depth / contact information |
| **02 Understanding** | Recognize semantics and relationships | Perception data | Objects / attributes / scene relationships |
| **03 Localization** | Determine target position | Understanding results | Box / mask / pose candidates |
| **04 Planning** | Decompose task into steps | Localization information | Subtasks / path / strategy |
| **05 Execution** | Control execution actions | Planning steps | Trajectory / control commands |
| **06 Feedback** | Closed-loop correction | Execution results | Error / state updates |

```
Perception → Understanding → Localization → Planning → Execution → Feedback → (back to Perception)
```

## 6. Embodied Perception: Four Types of Perceptual Targets

| Perception Type | Content | Key Technology | Typical Scenario |
|-----------------|---------|----------------|------------------|
| **Object Perception** | Geometry, articulated structures, physical properties | PointNet++, 3D detection | Grasping, manipulation |
| **Scene Perception** | Scene reconstruction and understanding | NeRF, 3D GS, Semantic SLAM | Navigation, exploration |
| **Behavior Perception** | Gestures, human pose, action understanding | Pose Estimation, Action Recognition | Human-robot collaboration |
| **Expression Perception** | Emotion, intent detection | Facial / Audio Analysis | Service robots |

### 6.1 3D Data Representations

| Data Format | Description | Source | Encoding Method |
|-------------|-------------|--------|-----------------|
| **Point Cloud** | A set of (x, y, z) coordinate points + features | LiDAR | PointNet, PointNet++ |
| **Mesh** | Points, edges, faces representing object surfaces | CAD, point cloud conversion | MeshNet |
| **Voxel** | Cubic grids | Point cloud conversion | VoxelNet, DeepSDF |
| **Depth Map** | Each pixel corresponds to a depth value | Stereo / structured light / ToF | GVCNN |

> PointNet is the classic method for point cloud encoding -- it uses MLPs to encode each point individually and then aggregates global features. PointNet++ addresses its difficulty in capturing local features by introducing hierarchical sampling.

|---------|-------|-----------------|

> Robot data is shifting from "single lab, single task" toward an open data ecosystem of "cross-scene, cross-platform, cross-embodiment."

## 8. Current Bottlenecks and Frontier Directions

### 8.1 Current Bottlenecks in Embodied Intelligence

| Bottleneck | Manifestation |
|------------|---------------|
| **Perception is already strong** | Multimodal LLMs like LLaVA can already perform multi-sensor integrated analysis |
| **Reasoning still lags** | Mainstream LLMs underperform on embodied reasoning benchmarks |
| **Execution faces fundamental difficulties** | Scenarios are too complex; building a complete skill library is nearly impossible |

> Therefore, current AI is not yet sufficient to make robots truly intelligent -- **embodied intelligence is needed.**

### 8.2 Frontier Research Directions

| Direction | Description | Representative Work |
|-----------|-------------|---------------------|
| **VLA Models** | Vision-Language-Action end-to-end models | Uni-Sight, RT-2 |
| **VLN Models** | Vision-Language-Navigation | --- |
| **Multi-Agent** | Automated programming, complex task interpretation | Hermes, OpenClaw |
| **Lightweighting** | Edge deployment of LLMs / embodied intelligence | Quantization / Pruning / Distillation |
| **Pretraining → Fine-tuning → Post-training** | Full pipeline for multimodal LLMs | --- |

## 9. Summary

```text
Evolution of Vision Models:
  Pixels → Handcrafted Features → CNN → ViT / Transformer → Vision Foundation Models
                                                              ↓
                                              CLIP (Image-Text Alignment)
                                                              ↓
                                        Grounding DINO (Open-Vocabulary Detection)
                                                              ↓
                                                SAM (General-Purpose Segmentation)
                                                              ↓
                                    Embodied VLA (Perception → Reasoning → Execution)

Embodied Intelligence = Vision Foundation Models + Language Understanding + Robot Execution + Closed-Loop Feedback
```

> Embodied intelligence in the era of foundation models is not simply about putting a large model into a robot. It requires vision foundation models to understand the environment, language models to understand tasks, VLA models to connect perception and action, and closed-loop feedback to ensure robustness. This is a systems engineering endeavor.
