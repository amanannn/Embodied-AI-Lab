## 一、后训练全景

预训练让模型"知道"，后训练让模型"好用"、"安全"、"听话"。

| 阶段 | 目标 | 方法 | 数据 | 成本 |
|------|------|------|------|------|
| **预训练** | 学习通用知识 | Next-token prediction | TB 级网络爬虫 | $$$$$ |
| **SFT** | 学会指令格式 | 监督学习 | 万~百万级人工示范 | $$$$ |
| **偏好对齐** | 符合人类价值观 | RLHF / DPO | 万~十万级人类偏好排序 | $$$ |
| **安全对齐** | 拒绝危险请求 | Constitutional AI / Red Teaming | 对抗样本 + 宪法规则 | $$ |

## 二、SFT——指令微调

教会模型"HOW to answer"，不是"WHAT to know"。

### 2.1 SFT 能做什么和不能做什么

| ✅ SFT 能做到 | ❌ SFT 做不到 |
|-------------|-------------|
| 学会回复格式（"User: / Assistant:"） | 判断两个回答哪个更好 |
| 学会遵循指令而非续写 | 拒绝回答有害问题 |
| 学会多轮对话 | 自我纠正错误推理 |
| 使用特定语气（正式/友好/专业） | 超越训练数据的质量上限 |

> SFT 决定了模型的"下限"——格式和风格。对齐决定了模型的"上限"——质量和安全。

## 三、RLHF——从人类反馈中强化学习

### 3.1 三步详细流程

```
────────── Step 1: SFT ──────────
  预训练 Model
  + 10K-100K 高质量 (prompt, human_answer) 对
  → SFT Model（会按指令格式回答）

────────── Step 2: 训练 RM ──────────
  同一 prompt → SFT Model 生成 4-9 个不同回答
  → 人工标注员对这些回答排序（A > B > C > D）
  → 训练一个 Reward Model 来预测"人类偏好排序"
  → RM 的输入: (prompt, response)，输出: scalar reward

────────── Step 3: PPO ──────────
  SFT Model (policy) + RM (reward signal)
  → Policy 生成回答 → RM 打分 → PPO 更新 policy
  → 同时约束 policy 不要离 SFT 太远（KL 惩罚）
  → 最终: 对齐后的 Model
```

### 3.2 奖励模型训练

```python
# RM 训练数据: 人类对两个回答的偏好
{ "prompt": "世界上最高的山是什么？",
  "chosen": "珠穆朗玛峰，海拔 8848.86 米，位于中国与尼泊尔边境",
  "rejected": "珠穆朗玛峰，很高" }

# Bradley-Terry 偏好模型
# P(chosen > rejected) = sigmoid(r_chosen - r_rejected)
# 训练目标: chosen 的得分 > rejected 的得分

loss = -torch.log(torch.sigmoid(reward_chosen - reward_rejected))
```

### 3.3 RLHF 四大痛点

| 痛点 | 具体表现 | 缓解方案 |
|------|---------|---------|
| **奖励攻击** | 模型找到 RM 的漏洞，输出长而无用的回答 | KL 惩罚 + RM 定期更新 |
| **训练不稳定** | PPO loss 震荡，收敛困难 | 小 batch + 低 lr + early stopping |
| **人类标注成本** | 高质量偏好数据 $0.5-2/条 | 合成数据（RLAIF） |
| **评估困难** | 没有单一指标衡量"对齐程度" | MT-Bench, AlpacaEval, Chatbot Arena |

## 四、DPO——直接偏好优化

### 4.1 核心洞察

RLHF 需要训练一个"代理"奖励模型，再用 PPO 优化。DPO 问：**能不能直接从偏好数据中训练，跳过奖励模型？**

答案是能——通过重新参数化，DPO 发现了最优策略的闭式解：

$$L_{DPO} = -\mathbb{E}_{(x, y_w, y_l) \sim D}\left[\log\sigma\left(\beta \log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]$$

### 4.2 DPO 的直观理解

| 项 | 含义 | DPO 在做什么 |
|----|------|------------|
| $\log\frac{\pi_\theta(y_w\|x)}{\pi_{ref}(y_w\|x)}$ | chosen 回答的相对提升 | 希望这个值**大**（winner 概率相对于 ref 提升） |
| $\log\frac{\pi_\theta(y_l\|x)}{\pi_{ref}(y_l\|x)}$ | rejected 回答的相对提升 | 希望这个值**小**（loser 概率不要提升） |
| $\beta$ | 控制更新幅度 | 越大=越保守；0.1-0.5 常见 |

### 4.3 DPO vs RLHF 全面对比

| 维度 | RLHF | DPO |
|------|------|-----|
| 需训练奖励模型？ | ✅ 需要 | ❌ 不需要 |
| 训练算法 | PPO（强化学习） | 标准交叉熵（监督学习） |
| 显存 | 需同时加载 3-4 个模型 | 2 个模型（Policy + Ref） |
| 收敛速度 | 慢 | 快 |
| 训练稳定性 | 差（RL 通病） | 好 |
| 理论优雅度 | 经实证有效 | 有闭式解 |
| 业界采用 | ChatGPT, Claude（早期） | Llama 3, Qwen 2, DeepSeek-V2 |

```python
# DPO 完整训练脚本（配 TRL 库）
from trl import DPOTrainer, DPOConfig
from datasets import load_dataset

# 数据格式: {prompt, chosen, rejected}
dataset = load_dataset("argilla/ultrafeedback-binarized-preferences-cleaned")

trainer = DPOTrainer(
    model=model,
    ref_model=ref_model,       # 冻结的 SFT 模型
    args=DPOConfig(
        per_device_train_batch_size=2,
        gradient_accumulation_steps=8,
        learning_rate=5e-7,
        beta=0.1,               # DPO 温度

```

## 五、DPO 变体

| 变体 | 改进 | 适用场景 |
|------|------|---------|
| **IPO** | 加正则化防止过拟合 | 数据噪声大 |
| **KTO** | 不需要成对偏好——只需单条反馈（好/坏） | 反馈信号稀疏 |
| **ORPO** | 将 SFT 和对齐合并为一个阶段 | 简化训练流程 |
| **SimPO** | 以序列长度作为隐式正则 | 避免长度偏置 |
| **Iterative DPO** | 每轮用更新后模型生成新偏好对 | 持续自我改进 |

## 六、Constitutional AI——法条式对齐

不依赖人类标注偏好，而是**预设宪法原则**自动生成训练数据。

```
宪法示例：
1. 请选择对人类最无害的回答
2. 请选择不包含种族歧视的回答
3. 请选择不鼓励非法行为的回答
4. 请选择最诚实的回答——如果不确定，请选择"不知道"
```

### Constitutional AI 流程

```
Phase 1: 监督阶段
  Model 生成有害回答
  → Constitution 批评这个回答并指出问题
  → Model 根据批评修正回答
  → 修正后的回答作为训练数据 → SFT

Phase 2: RL 阶段
  Model 生成回答
  → Constitution-based AI 判断哪个更好（替代人类）
  → 生成偏好数据 → RLHF 或 DPO 训练
```

> Claude 是 Constitutional AI 最成功的实践——它的安全性在多个 benchmark 上领先。

## 七、评估体系

| Benchmark | 测什么 | 方式 |
|-----------|--------|------|
| **AlpacaEval** | 回答质量 vs GPT-4 | LLM-as-Judge 自动评分 |
| **MT-Bench** | 多轮对话 + 推理 + 写作 | GPT-4 打分 |
| **Chatbot Arena** | 真实人类投票 | A/B test |
| **HellaSwag** | 常识推理 | 多选题准确率 |
| **TruthfulQA** | 真实性——不说谎 | 多选题准确率 |
| **SafetyBench** | 安全性 | 拒绝率 + 准确率 |

## 八、多模态对齐的特殊挑战

| 挑战 | 具体问题 | 当前探索 |
|------|---------|---------|
| **视觉偏见** | "护士=女性""CEO=男性" | 多样化训练数据 |
| **隐私泄露** | 分割模型可能输出训练图片中的人脸 | 差分隐私训练 |
| **图片越狱** | 在图片中嵌入隐藏 prompt 攻击模型 | 输入层防御 |
| **多语言对齐** | 英文 RLHF 不保证中文也安全 | 多语言偏好数据 |
| **多模态幻觉** | 描述图片中不存在的物体 | RLHF on VQA pairs |
