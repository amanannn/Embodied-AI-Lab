## 一、后训练：预训练到产品的最后一公里

预训练让模型"知道"，后训练让模型"好用"。

| 阶段 | 目标 | 方法 |
|------|------|------|
| **预训练** | 学习通用知识 | Next-token prediction |
| **SFT** | 学会指令格式 | 人工标注的指令-回答对 |
| **RLHF** | 学会人类偏好 | 从人类反馈中强化学习 |
| **对齐** | 学会价值判断 | 安全、诚实、有用 |

---

## 二、RLHF——三步流程

```
Step 1: SFT
  预训练模型 + 高质量人类示范 → SFT 模型

Step 2: 奖励模型（RM）
  同一 prompt → 多个回答 → 人类排序
  → 用模型预测"人类更喜欢哪个"

Step 3: PPO 强化学习
  SFT 模型 + RM 评分 → PPO 优化
  → 最大化 RM 分数，同时不偏离 SFT 太远
```

### 奖励模型训练

```python
# 训练数据格式
{
  "prompt": "如何做番茄炒蛋？",
  "chosen": "先放油，再放鸡蛋...",   # 人类偏好
  "rejected": "番茄炒蛋的做法是..."   # 人类不偏好
}
# 目标：chosen 分数 > rejected 分数
loss = -log(sigmoid(score_chosen - score_rejected))
```

### RLHF 局限

| 问题 | 说明 |
|------|------|
| **成本高** | 大量人类标注排序 |
| **奖励攻击** | 模型迎合 RM 而非真正变好 |
| **训练难** | PPO 出名难调 |
| **文化偏差** | 标注员偏好不具普遍性 |

## 三、DPO——直接偏好优化

不需要奖励模型，直接从偏好数据中优化。

$$L_{DPO} = -\mathbb{E}\left[\log\sigma\left(\beta \log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]$$

| 符号 | 含义 |
|------|------|
| $y_w$ | 人类偏好的回答（winner） |
| $y_l$ | 人类不喜欢的回答（loser） |
| $\pi_\theta$ | 训练中的模型 |
| $\pi_{ref}$ | 冻结的参考模型 |
| $\beta$ | 控制偏离程度 |

### DPO vs RLHF

| 维度 | RLHF | DPO |
|------|------|-----|
| 组件 | Policy + RM + PPO | 仅 Policy |
| 训练难度 | 高 | 低——标准监督学习 |
| 显存 | 3-4 个模型 | 2 个 |
| 效果 | 理论更高 | 实践相当 |

```python
def dpo_loss(pi_logps, ref_logps, yw_idx, yl_idx, beta=0.1):
    pi_ratio_w = pi_logps[yw_idx] - ref_logps[yw_idx]
    pi_ratio_l = pi_logps[yl_idx] - ref_logps[yl_idx]
    loss = -torch.log(torch.sigmoid(beta * (pi_ratio_w - pi_ratio_l)))
    return loss.mean()
```

## 四、对齐——让 AI 符合人类价值

### 4.1 三个目标

| 目标 | 含义 | 反例 |
|------|------|------|
| **Helpful** | 回答实际需求 | 答非所问 |
| **Honest** | 不知道就说不知道 | 编造论文引用 |
| **Harmless** | 拒绝危险请求 | 教人制作炸弹 |

### 4.2 对齐方法

| 方法 | 原理 | 代表 |
|------|------|------|
| **RLHF** | 人类排序→RM→PPO | ChatGPT |
| **DPO** | 直接从偏好对中学习 | Llama 3, Mistral |
| **Constitutional AI** | 预设宪法原则自动生成训练数据 | Claude |
| **RLAIF** | AI 替代人类做判断 | Anthropic |
| **Red Teaming** | 专门找人攻击模型漏洞 | OpenAI, Meta |

#### Constitutional AI 流程

```
1. 生成回答
   Model 生成有害回答 → Constitution 指出问题 → Model 修正

2. 生成偏好数据
   修正后的回答 = chosen，原有害回答 = rejected

3. 训练
   用 generated 偏好数据做 RLHF 或 DPO
```

---

## 五、多模态对齐的特殊挑战

| 挑战 | 说明 |
|------|------|
| **视觉偏见** | 学到"银行家=白人男性"等刻板印象 |
| **隐私** | 视觉模型可能泄露训练数据中的人脸 |
| **安全性** | 防止通过图片注入越狱 prompt |
| **多语言对齐** | 英语对齐不一定迁移到中文 |

---

## 六、对齐的未来

```text
当前主流：
  人工标注偏好 → RLHF/DPO → 对齐后的模型

新趋势：
  1. 合成数据——AI 生成偏好数据，降低成本
  2. 个性化对齐——不同用户有不同的价值观
  3. 可逆对齐——允许查看对齐前的原始输出
  4. 持续对齐——部署后持续收集反馈并更新
```
