## 一、多模态大模型基本概念

### 1.1 大模型的核心演进：PLM vs LLM

| 维度 | PLM（预训练语言模型） | LLM（大语言模型） |
|------|---------------------|-------------------|
| **参数规模** | 百万～十亿级 | 十亿～千亿级+ |
| **代表模型** | BERT（双向编码）、GPT-1/2（自回归） | GPT-3（1750亿）、GPT-4（~1.8万亿）、PaLM 2（3400亿） |
| **核心能力** | 迁移学习、微调适配下游任务 | 跨任务泛化、指令遵循、涌现能力 |

### 1.2 关键核心规律与效应

#### 涌现能力（Emergence）

当模型规模达到某个特定临界阈值时，会**突然展现出小模型不具备的非线性提升能力**。

| 三大典型表现 | 说明 |
|------------|------|
| **上下文学习（In-context Learning）** | 仅通过少数示例即可完成全新任务 |
| **思维链推理（Chain-of-Thought）** | 多步骤逻辑推理，逐步拆解复杂问题 |
| **指令遵循（Instruction Following）** | 准确理解并遵循复杂的自然语言指令 |

> 出现机理：模型规模扩大导致表征能力跃升；海量数据使更多知识与模式被捕获；复杂能力在大规模下被激活。

#### 规模定律（Scaling Law）

LLM 的性能（Test Loss）随着**模型参数规模、训练数据量、计算资源**的增加，呈现可预测的持续提升。

| 三要素 | 说明 |
|--------|------|
| 模型参数规模（Model Size） | 越大越强，但需数据匹配 |
| 训练数据量（Data Size / Tokens） | 模型越大需要的数据越多 |
| 计算资源（Compute） | GPU 小时数的平方级增长 |

> 数据瓶颈：模型参数与数据规模必须**协同扩展**才能持续获得性能提升。

### 1.3 五大范式转变

| 范式转变方向 | 从（传统模式） | 到（大模型模式） | 核心目标 |
|-------------|--------------|----------------|---------|
| **单模态 → 多模态** | 单一文本处理 / 专用 CV | 图、文、音、视频深度融合 | 跨模态全面理解与推理 |
| **预测 → 生成** | 判别式任务（分类/判断） | 生成式创造（创作/生成） | 开启 AIGC 时代 |
| **单任务 → 多任务** | 为每个任务训练专用模型 | 一个通用模型胜任多种任务 | 统一接口，降低门槛 |
| **感知 → 认知** | 视觉/语音等"识别"层 | 关系理解、逻辑推理决策层 | 从"看见"到"看懂为什么" |
| **大模型 → 智能体** | 被动响应式助手（Copilot） | 主动执行智能体（AI Agent） | 目标→规划→工具→执行→完成 |

### 1.4 多模态基础模型的三大核心方向

| 方向 | 定位 | 代表模型 | 典型任务 |
|------|------|---------|---------|
| **视觉理解类** | 从像素到语义深层理解 | GPT-4V, LLaVA, BLIP-2, MiniGPT-4 | 图像描述、VQA、图文检索 |
| **视觉生成类** | 学习数据分布，生成高质量内容 | DALL-E 3, Stable Diffusion, Midjourney, Sora | 文生图、图生图、文生视频 |
| **通用接口类** | 统一多模态输入，连接理解/生成/行动 | GPT-4 API, LangChain, AutoGPT | Agent 编排与工具调用 |

## 二、代表性技术详解：BERT vs ViT vs MAE

### 2.1 BERT——NLP 预训练里程碑

Google 2018 年提出，基于 Transformer 双向编码，开创**"预训练 + 微调"**范式。

| 规格 | 层数 | 参数量 |
|------|------|--------|
| BERT-Base | 12 层 Transformer 编码器 | 1.1 亿（110M） |
| BERT-Large | 24 层 Transformer 编码器 | 3.4 亿（340M） |

#### 输入表示构建

$$\text{Input Embedding} = \text{Token 嵌入} + \text{Segment 嵌入} + \text{Position 嵌入}$$

| 嵌入类型 | 说明 |
|---------|------|
| **Token 嵌入** | 词/子词通过可训练词向量矩阵映射为稠密向量（维度 768） |
| **Segment 嵌入** | 区分不同句子（$E_A$ / $E_B$），服务 NSP 任务 |
| **Position 嵌入** | 可学习的绝对位置编码（最大长度 512） |

#### 两大预训练任务

| 任务 | 机制 | 损失 |
|------|------|------|
| **MLM（掩码语言模型）** | 随机选 15% Token：80%→`[MASK]`、10%→随机词、10%→保留原词 | 仅对被遮盖位置计算交叉熵 |
| **NSP（下一句预测）** | `[CLS]` 汇聚句对信息 → 二分类判断 B 是否为 A 的下一句（正负 1:1） | 句级交叉熵 |

> MLM 的 80-10-10 策略本质是缩小预训练与微调阶段的输入不一致——微调时下游数据没有 `[MASK]` 标记。

### 2.2 ViT——CV 进入 Transformer 时代

#### 核心挑战与方案

Transformer 直接用于图像会导致序列过长：$O(n^2)$ 自注意力在像素级不可接受。

**切块策略（Patching）**：将 $224 \times 224$ 图像切成 $16 \times 16$ 的 Patch，每个 Patch 拉平为 768 维向量，形成 196 个"视觉 Token"+ 一个可学习的 `[class]` 编码。

| 对比维度 | CNN | ViT |
|---------|-----|-----|
| 感受野 | 局部→逐层扩大 | 全局自注意力直接建模 |
| 归纳偏置 | 内置局部性 + 平移不变性 | 无，需大量数据学习 |
| 小数据集表现 | 较好 | 可能不如 CNN |
| 大数据集表现 | 饱和 | 持续提升，超越 CNN |

#### 预训练任务——掩码图像块预测（MPP）

随机遮盖部分 Patch，在像素或特征空间预测重建被遮盖内容，最小化重建误差（MSE/L1）。

### 2.3 MAE——掩码自编码器

何恺明 2021 年提出。非对称架构：**编码器重（标准 ViT），解码器轻**（层数少、参数小）。

| 特性 | 说明 |
|------|------|
| **极高掩码率** | **75%**——只保留 25% 可见 Patch 送入编码器 |
| **设计意图** | 强制模型通过极其有限的信息学习深层全局语义 |
| **解码器** | 预训练时处理包含掩码 Token 的全量 Patch 做像素级重建；推理时直接丢弃 |

#### 视觉（MAE）与语言（BERT）预训练核心差异

| 维度 | BERT（语言） | MAE（视觉） |
|------|------------|-----------|
| **掩码率** | 15%（低） | 75%（极高） |
| **信息密度** | 文本语义丰富、冗余低 | 图像像素级冗余度极高 |
| **任务类型** | 词级预测（分类） | 像素级重建（回归） |
| **解码器** | 任务头起辅助作用 | 轻量解码器是重建核心 |

## 三、预训练机制

### 3.1 预训练发展演进

| 时期 | 阶段 | 代表技术 | 特点 |
|------|------|---------|------|
| 2013-2015 | 静态词向量 | Word2Vec, GloVe, FastText | 基于统计，无法建模上下文 |
| 2016-2017 | 动态上下文 | ELMO, CoVe, Seq2Seq | 双向 LSTM，词表示随上下文变化 |
| **2017** | **分水岭** | **Transformer** | Attention Is All You Need——并行化 + 长距离依赖 |
| 2018-2019 | 预训练+微调 | BERT, GPT | 通用表征 + 迁移学习成为主流 |
| 2020-2022 | 大规模基础模型 | GPT-3, CLIP, DALL-E, MAE | 千亿参数 + 上下文学习 |
| 2023+ | 统一多模态 | GPT-4V, Gemini, PaLM-E | 多元模态通过统一架构走向大一统 |

### 3.2 为什么需要预训练？

| 原因 | 说明 |
|------|------|
| **解决表示空间差异** | 文本（离散符号）、图像（连续像素）、语音的底层分布各异，预训练学出统一跨模态语义嵌入空间 |
| **缓解极高标注成本** | 多模态数据人工标注成本指数级增长，预训练利用海量无标注数据自监督学习 |
| **建立跨模态对齐与复用** | 先学"通识教育"（通用表示），后学"专业教育"（下游微调） |

### 3.3 预训练的核心价值

| 价值 | 量化效果 |
|------|---------|
| **数据效率** | 下游标注数据需求降低 10～100 倍 |
| **收敛速度** | 微调收敛速度提升 5～10 倍 |
| **性能飞跃** | 下游准确率平均提升 20% 以上 |

**两阶段标准流程**：

```
阶段 1：大规模无标注/弱标注数据预训练
    → 自监督任务自动生成监督信号
    → 产出具备通用表示能力的基础底座模型

阶段 2：小规模任务数据微调
    → 在预训练权重基础上继续优化
    → 更新参数快速适配特定应用
```

## 四、NLP 核心预训练任务

| 预训练任务 | 机制 | 训练信号 | 输入示例 | 核心能力 | 代表模型 |
|-----------|------|---------|---------|---------|---------|
| **MLM**（掩码语言模型） | 随机遮蔽 15% Token，双向上下文预测 | 词级多分类交叉熵 | `我今天去 [MASK] 馆学习` | 双向上下文建模 | BERT, RoBERTa, ALBERT |
| **DAE**（去噪自编码器） | 加噪声（删除/打乱/替换），训练去噪重构 | 序列重构交叉熵 | 带噪声输入 → 完整原文 | 鲁棒特征表示 | BART, T5, SpanBERT |
| **RTD**（替换词检测） | 小生成器替换部分词，大判别器判断每个词真假 | 二分类交叉熵 | 标签：`[真,真,真,假,真,假]` | 100% 样本利用，细粒度语义区分 | ELECTRA, DeBERTa |
| **NSP**（下一句预测） | 判断 B 是否为 A 的逻辑后续句 | 句级二分类 | `[CLS] A [SEP] B [SEP]` | 句间连贯性（任务偏简单） | BERT 原始版 |
| **SOP**（句子顺序预测） | NSP 挑战替代版——判断两片段顺序是否正确 | 句级二分类 | 正：A→B / 负：B→A | 深度理解句间逻辑 | ALBERT |

> RTD 用判别任务替代生成任务，100% 样本利用率；SOP 成功避免主题泄露，迫使模型理解承接与因果关系。

## 五、CV 预训练任务体系

### 5.1 四大类别

| 类别 | 机制 | 代表方法 |
|------|------|---------|
| **特定代理任务** | 利用图像结构设计自监督信号 | 拼图任务（Jigsaw）、旋转预测（RotNet）、上色、修补 |
| **帧序学习** | 利用视频时序连贯性 | 未来帧预测、时序前后判定（Temporal Order）、CPC |
| **生成与重建** | 拟合数据分布或重建损坏区域 | GAN, VAE, 扩散模型 / MIM（MAE, BEIT, SimMIM） |
| **对比学习** | 正样本靠近、负样本远离 | SimCLR, MoCo, BYOL, SWAV, DINO |

## 六、前沿架构与技术演进

### 6.1 GPT 系列：从微调到上下文学习

| 版本 | 年份 | 核心贡献 |
|------|------|---------|
| **GPT-1** | 2018 | 无监督自回归预训练 + 有监督微调；掩码多头自注意力 |
| **GPT-2** | 2019 | 取消任务微调，统一无监督多任务 |
| **GPT-3** | 2020 | 1750 亿参数；上下文学习——无需梯度更新，Prompt + Few-shot 执行多样化任务 |
| **GPT-4** | 2023 | 多模态 Transformer，原生文字+图像，RLHF 价值对齐 |
| **GPT-5（预期）** | — | 多模态 MoE 稀疏架构，更长上下文，超级 Agent 自主规划 |

### 6.2 稀疏专家混合架构（MoE）

| 组件 | 功能 |
|------|------|
| **门控网络（Router）** | 每个 Token 输入时，Softmax 计算权重 |
| **稀疏激活** | 仅激活 Top-k（通常 k=2）个专家参与计算 |
| **输出** | 加权求和 $y = \sum g_i(x)E_i(x)$ |

> 在不增加单次推理计算量的前提下，实现 10 倍+ 的模型参数容量扩展。

### 6.3 DeepSeek 效率革命

| 创新 | 机制 | 效果 |
|------|------|------|
| **DeepSeekMoE** | 极致专家特化稀疏激活 | 万亿参数，极低计算成本 |
| **MLA 注意力** | K 和 V 低维联合压缩（Latent Vector） | 大幅降低 KV Cache 内存，打破长文本推理硬件瓶颈 |
| **稀疏注意力** | $O(L^2)$ → $O(L \cdot k)$ 线性复杂度 | 长序列高效处理 |

## 七、思考题：结合 BERT 和 ViT 设计多模态模型

参考 **CLIP（Contrastive Language-Image Pre-training）** 的经典范式：

| 步骤 | 组件 | 作用 |
|------|------|------|
| 1 | **文本编码器（BERT-based）** | 双向编码文本序列，提取句级/词级语义向量 |
| 2 | **视觉编码器（ViT-based）** | 图像切 Patch → Transformer，提取全局视觉特征 |
| 3 | **跨模态对比学习对齐** | 统一映射层 → 图文特征投影到同一共享空间 |
| 训练 | 正样本靠近 / 负样本远离 | 最大化匹配图文对相似度 |
| 应用 | 跨模态检索 / Zero-shot 分类 | 文搜图、图搜文、零样本泛化 |

> CLIP 的核心不是架构多么创新，而是用**4 亿图文对**证明了：足够大的数据和足够简单的目标（对比学习），可以学到通用的跨模态表示。
