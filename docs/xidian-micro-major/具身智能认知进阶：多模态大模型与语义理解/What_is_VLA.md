## 一、VLA 技术起源

### 1.1 VLA 是什么？

VLA（Vision-Language-Action）是具身智能的核心模型范式——让机器人同时"看懂"环境、"听懂"指令、"做出"动作：

| 组件 | 作用 | 形式 |
|------|------|------|
| **V（Vision）** | 理解环境 | RGB 图像 / 深度图 / 点云 |
| **L（Language）** | 理解任务 | 自然语言指令 |
| **A（Action）** | 产生动作 | 关节角度 / 末端位姿 / 控制命令 |

$$A = f_\theta(V, L)$$

> VLA = 让机器人"看懂"（Vision）、"听懂"（Language）、"会做"（Action）。

### 1.2 为什么需要 VLA？——传统范式的终结

#### 传统机器人编程的困境

| 传统方式 | 问题 |
|---------|------|
| 手工编程每个动作 | 换任务 = 重写代码 |
| 固定技能库 | 场景太复杂，无法穷举 |
| 示教编程 | 需要人类一步步演示 |
| 模块化流水线 | 感知→规划→控制，误差逐级放大 |

传统机器人的核心矛盾：**结构化编程 vs. 开放世界**。真实环境有无穷多的物体、场景和任务组合——代码永远写不完。

#### VLA 的突破

| 能力 | 传统 | VLA |
|------|------|-----|
| 任务指令 | 程序/API | **自然语言** |
| 场景适应 | 固定环境 | **泛化到新场景** |
| 新物体 | 需重新标注 | **Zero-shot 理解** |
| 动作生成 | 预编程 | **端到端学习** |
| 知识迁移 | 无 | **互联网规模预训练知识** |

### 1.3 技术演进路线：从模块化到端到端

VLA 并非凭空出现——它是视觉-语言模型（VLM）向具身智能的自然延伸：

```
传统机器人（模块化）
    Perception → Planning → Control
         ↓
视觉-语言模型（VLM）
    CLIP (2021) → BLIP-2 (2023) → LLaVA (2023)
         ↓
VLA 先驱探索
    SayCan (2022)     —— LLM 做高层规划，技能库执行
    PaLM-E (2023)     —— 多模态大模型直接融入视觉与机器人状态
    RT-1 (2022)       —— Transformer 做端到端视觉-动作映射
    Gato (2022)       —— 单一 Transformer 通吃多种任务（含机器人控制）
         ↓
VLA 范式确立
    RT-2 (2023)       —— "动作即 token"——VLM + 动作离散化
    Octo / OpenVLA (2024) —— 开源 VLA 基座
    Pi0 (2024)        —— 扩散动作 + 流匹配
```

**关键转折点：**
- **SayCan** 证明了 LLM 可以做机器人任务规划（但动作仍需预编程技能库）
- **RT-1** 证明了 Transformer 可以直接输出动作（但依赖特定机器人数据）
- **RT-2** 将两者统一——用同一个 Transformer 既理解语言又输出动作，真正实现"端到端 VLA"

### 1.4 三种基础动作生成范式

| 范式 | 动作表示 | 代表模型 | 核心思想 |
|------|---------|---------|---------|
| **离散化** | 动作空间离散为 token | RT-2, Octo | 复用 LLM 的 next-token prediction——"把动作当词语" |
| **扩散** | 去噪生成连续动作 | Diffusion Policy, Pi0 | 处理多模态动作分布——适合灵巧操作 |
| **回归** | 直接预测连续动作值 | ACT, BC | 简单直接，训练高效 |

> 这三种范式构成了后续所有先进 VLA 基座的技术底色。

## 二、VLA 先进基座技术

### 2.1 通用架构：四层堆叠

现代 VLA 基座模型普遍遵循四层架构：

```
视觉输入（相机）                    语言指令（文本）
      ↓                                  ↓
┌─────────────────────┐    ┌─────────────────────┐
│  Vision Encoder     │    │  Language Encoder   │
│  ViT / SigLIP /     │    │  LLaMA / Gemma /    │
│  DINOv2 / CLIP      │    │  Qwen / InternLM    │
└─────────┬───────────┘    └─────────┬───────────┘
          ↓                          ↓
          └────────────┬─────────────┘
                       ↓
          ┌─────────────────────────┐
          │   Multimodal Fusion     │
          │  Cross-Attention /      │
          │  Concat / Q-Former /    │
          │  MLP Projector          │
          └─────────────┬───────────┘
                        ↓
          ┌─────────────────────────┐
          │    Action Decoder       │
          │  Diffusion Head /       │
          │  Autoregressive /       │
          │  Flow Matching          │
          └─────────────┬───────────┘
                        ↓
                   动作输出
          (关节角度 / 末端位姿 / 轨迹)
```

### 2.2 代表模型深度解析

#### 全景对比

| 模型 | 年份 | 机构 | 核心思路 | 参数规模 | 开源 |
|------|------|------|---------|---------|------|
| **RT-2** | 2023 | Google DeepMind | VLM + 动作 token 化 | 55B | ✗ |
| **Octo** | 2024 | UC Berkeley | 开源通用策略，多本体多任务 | 27M–93M | ✓ |
| **OpenVLA** | 2024 | Stanford | 7B 开源 VLA，Llama + DINOv2 + SigLIP | 7B | ✓ |
| **Pi0（π₀）** | 2024 | Physical Intelligence | 扩散动作 + 流匹配 | 3B | ✗ |
| **GR00T N1** | 2024–25 | NVIDIA | 人形机器人基础模型 | ~2B | ✓ |
| **RDT-1B** | 2024 | 清华 | 1B 参数扩散 VLA，双手操作 | 1B | ✓ |

#### RT-2（Google DeepMind, 2023）

**核心思想：把动作当成"语言"**

```
"Pick up the red cup on the left"
        ↓
PaLI-X / PaLM-E（视觉-语言骨干，55B）
        ↓
输出 token 序列：
    "<grip> 128 256 45 <release> 320 180 90"
        ↓
Token → 关节角度 / 末端位姿 → 机器人执行
```

**技术要点：**
- **动作 tokenization**：将连续 6-DoF 动作离散化为 256 个 bin，每个 bin 对应一个 token
- **Co-fine-tuning**：在互联网图文数据 + 机器人数据上联合微调
- **关键发现**：大规模 VLM 预训练提供的语义理解能直接迁移到机器人控制任务

> RT-2 的核心洞察：**不需要专门的"动作网络"**——如果一个模型能理解"抓取红色杯子"的语义，它也能学会对应的动作 token 序列。

---

#### Octo（UC Berkeley, 2024）

**核心思想：一个模型驾驭多种机器人**

| 设计目标 | 实现方式 |
|---------|---------|
| **多本体支持** | 输入包含本体描述（自由度、相机参数），模型学习本体无关表征 |
| **多任务统一** | 支持 25+ 数据集联合训练 |
| **轻量开源** | 最小 27M 参数，可在消费级 GPU 上微调 |
| **即插即用** | 提供 HuggingFace 模型 + 标准推理接口 |

**架构：**
- 视觉编码器：轻量 ResNet / ViT
- 语言编码器：T5-base
- 动作头：Transformer decoder + 扩散头
- 训练数据：Open X-Embodiment 全量数据（100 万+ 轨迹）

> Octo 的意义：证明了**开源 VLA 基座模型是可行的**——不再只有 DeepMind 能做。

---

#### OpenVLA（Stanford, 2024）

**核心思想：用最强的开源组件拼出最强的开源 VLA**

```
视觉输入（多视角 RGB）           语言指令
      ↓                             ↓
┌──────────────┐          ┌──────────────┐
│ SigLIP (ViT) │          │   LLaMA-7B   │
│  + DINOv2    │          │  (language)   │
│  (vision)    │          │               │
└──────┬───────┘          └──────┬────────┘
       ↓                         ↓
       └──────────┬──────────────┘
                  ↓
         ┌───────────────┐
         │  MLP Projector│
         │  (fusion)     │
         └───────┬───────┘
                 ↓
         ┌───────────────┐
         │  Action Head  │
         │  (离散化 token)│
         └───────────────┘
                 ↓
            动作输出
```

**关键创新：**
- **双视觉编码器融合**：SigLIP（语义理解）+ DINOv2（空间理解）互补
- **高效微调**：在 Open X-Embodiment 上用 LoRA 微调，仅更新 ~1% 参数
- **性能**：在 LIBERO benchmark 上接近 RT-2-X（55B），仅 7B 参数

---

#### Pi0 / π₀（Physical Intelligence, 2024）

**核心思想：扩散模型是机器人动作生成的最佳选择**

| 组件 | 选型 | 作用 |
|------|------|------|
| **视觉骨干** | PaliGemma | 图像 → 多模态特征 |
| **语言骨干** | Gemma | 指令 → 文本特征 |
| **扩散动作头** | Flow Matching | 从噪声中迭代生成连续动作序列 |
| **输出频率** | 50Hz | 满足灵巧操作实时性需求 |

**为什么用扩散？**
- 机器人动作分布往往是**多模态**的（同一个任务可以有多种抓取方式）
- 扩散模型天然适合建模多模态分布——而回归模型只能学"平均动作"，导致执行失败
- **流匹配（Flow Matching）** 比传统 DDPM 采样快 10×，使实时控制成为可能

> "The GPT moment for robotics"——Pi0 被广泛视为机器人基础模型的 GPT-3 时刻。

---

#### GR00T N1（NVIDIA, 2024–2025）

**核心思想：人形机器人的通用基础模型**

| 特性 | 说明 |
|------|------|
| **目标硬件** | 人形机器人（双足/双臂） |
| **训练数据** | 合成数据 + 遥操作数据 + 互联网视频 |
| **仿真平台** | Isaac Sim / Isaac Lab |
| **开源** | 2025 年开源模型权重与训练框架 |
| **关键能力** | 全身协调控制、双手操作、长序列任务 |

**技术栈：**
- Vision-Language 骨干：基于 NVIDIA VILA 架构
- 动作生成：扩散 Transformer（DiT）
- 训练策略：仿真大规模预训练 → 真实机器人微调（Sim-to-Real）

---

#### RDT-1B（清华, 2024）

**核心思想：扩散 Transformer + 大规模参数 = 灵巧双手**

- 1B 参数扩散模型，专为**双手协调操作**设计
- 在 46 个数据集上联合训练
- 证明了 scaling law 在机器人扩散策略中成立：更大模型 → 更强泛化

### 2.3 技术路线分化总结

```
VLA 基座技术树
├── 离散化路线（token-based）
│   ├── RT-2 (2023)     —— 奠基石，闭源
│   ├── Octo (2024)     —— 轻量开源
│   └── OpenVLA (2024)  —— 7B 开源标杆
│
├── 扩散路线（diffusion-based）
│   ├── Diffusion Policy (2023) —— 学术奠基
│   ├── Pi0 (2024)      —— "GPT 时刻"
│   ├── RDT-1B (2024)   —— 双手操作 scaling
│   └── GR00T N1 (2025) —— 人形机器人
│
└── 融合路线
    └── 未来趋势：离散 token 作为高层规划 + 扩散作为底层执行
```

## 四、VLA 先进改进技术

### 4.1 推理增强 VLA：从"反射"到"思考"

传统 VLA 的问题是"感知→动作"近乎反射式——缺乏中间推理环节。当任务需要多步规划、计数、空间关系推理时，往往失败。

#### 技术路线对比

| 方法 | 核心思想 | 代表工作 | 效果 |
|------|---------|---------|------|
| **Embodied CoT** | 在动作前生成推理链，显式输出"思考过程" | ECoT (2024), ReVLA (2025) | 多步任务成功 +25% |
| **Inner Monologue** | 将环境反馈注入推理循环，允许模型自我纠错 | Inner Monologue (2022), Code-as-Policies (2023) | 动态重规划 |
| **ReAct-style VLA** | 交替进行"推理→观察→动作"，步进式决策 | RT-2-Reason (2024), Helix (Figure, 2025) | 复杂指令遵循 |

#### ECoT（Embodied Chain-of-Thought）详解

```
传统 VLA：
    "把蓝色方块放在红色方块右边"
        → [动作序列]

ECoT VLA：
    "把蓝色方块放在红色方块右边"
        → 推理：红色方块在（x=0.3, y=0.1），蓝色方块在（x=0.5, y=0.4）
        → 子任务1：抓取蓝色方块
        → 子任务2：移动到目标位置（x=0.4, y=0.1）
        → 子任务3：放置
        → [动作序列]
```

**关键设计：**
- **推理 token + 动作 token 交织**：模型先输出自然语言推理，再输出动作 token
- **多模态 CoT 数据合成**：用 GPT-4V 为机器人数据自动标注推理链
- **推理压缩**：推理过程不增加推理延迟——推理与动作并行编码

> 核心洞察：**让 VLA "说出它在想什么"**——推理链不仅提高成功率，还能让人类理解模型为什么会做出某个动作。

#### Helix（Figure, 2025）

Figure 人形机器人的商用 VLA 系统，采用**双模型架构**：

```
系统1（高频 ~200Hz）        系统2（低频 ~7Hz）
  "小脑"                       "大脑"
  │                            │
  快速反应性控制              语义理解 + 推理 + 规划
  (80M 参数 ViT + MLP)        (7B 参数 VLM)
  │                            │
  └──────────┬─────────────────┘
             ↓
      Latent 空间融合
             ↓
        动作输出（全身协调）
```

**关键创新：**
- 高频-低频解耦：灵巧操作需要 200Hz 频率，大模型推理只需 7Hz
- Latent 条件注入：系统 2 的语义意图以 latent vector 形式注入系统 1，而非逐 token 传递
- **全身协调**：统一控制双臂 + 双手 + 躯干 + 头部

### 4.2 空间与 3D VLA：从 2D 走向世界

#### 为什么需要 3D？

2D VLA 的根本局限：在桌面上方 10cm 处抓取物体，是一个**需要深度信息的三维空间问题**，而 2D 图像丢失了 z 轴。

| 方法 | 输入 | 空间表征 | 代表 |
|------|------|---------|------|
| **2D+深度** | RGB-D | 2.5D 特征图（像素 + 深度通道） | CLIPort, PerAct |
| **点云 VLA** | 3D 点云 | PointNet++ / 3D Transformer | 3D-ViTAC, Act3D |
| **3D 场景图** | 多视角 RGB→NeRF/3DGS | 显式 3D 特征场 | SpatialVLA, RoboPoint |
| **多视角融合** | 2-4 个 RGB 相机 | 跨视角 attention | RVT, OpenVLA (multi-view) |

#### SpatialVLA（2024–2025）

```
RGB 多视角图像
      ↓
┌──────────────────────┐
│  NeRF / 3D Gaussian  │
│  场景重建 + 特征蒸馏  │
└──────────┬───────────┘
           ↓
    3D 特征场（显式空间坐标）
           ↓
┌──────────────────────┐
│  Spatial Grounding   │
│  "红色方块在蓝色方块  │
│   左边 5cm 处"        │
└──────────┬───────────┘
           ↓
      动作（6-DoF 末端位姿）
```

**关键能力：**
- **空间关系推理**：理解"左边"、"上面"、"之间"等空间介词
- **度量操作**：精确执行"移动 5cm"、"旋转 30°"等定量指令
- **遮挡推理**：即使目标部分被遮挡，3D 表征仍能推断完整位姿

### 4.3 动作生成前沿：更快、更准、更稳

#### 从 DDPM 到 Consistency Model

| 方法 | 采样步数 | 推理延迟 | 动作质量 |
|------|---------|---------|---------|
| DDPM（经典扩散） | 100–1000 | ~1s | 最高 |
| DDIM | 10–100 | ~100ms | 高 |
| Flow Matching（Pi0） | 5–10 | ~50ms | 高 |
| **Consistency Model** | **1–2** | **<10ms** | 高（逼近 DDPM） |
| 自回归（RT-2） | 1（逐 token） | ~20ms | 中 |

#### Diffusion Forcing（2024–2025）

传统扩散策略的痛点：生成的是一段**固定长度的未来动作序列**，无法灵活处理需要"做多长时间不确定"的任务。

**Diffusion Forcing** 解决方案：
- 每个时间步只去噪一个动作 token，基于之前已确定的动作 history
- 支持**可变长度**的动作序列——"做完了就停"
- 结合扩散质量 + 自回归灵活性

```
扩散策略（传统）：        [a₁ a₂ ... aₙ]  ← 一次性去噪 N 步
Diffusion Forcing：       a₁ → a₂ → a₃ → ... ← 逐步去噪，可随时终止
```

#### 层级动作生成

```
层级 VLA 动作架构
├── 高层：任务规划（Task-level）
│   输出：子任务序列
│   "打开抽屉 → 取出物体 → 关上抽屉"
│   频率：~1Hz，基于 LLM
│
├── 中层：技能执行（Skill-level）
│   输出：末端位姿轨迹关键点
│   频率：~10Hz，基于扩散策略
│
└── 底层：运动控制（Motion-level）
    输出：关节角度 / 力矩
    频率：~100–1000Hz，基于经典控制器或学习策略
```

> 当前趋势：**不是用一个模型通吃所有频率，而是让不同层级用最适合的技术**。

### 4.4 高效适应：小数据撬动大模型

#### VLA 微调技术栈

| 技术 | 可训参数 | 典型性能 | 适用场景 |
|------|---------|---------|---------|
| **Full Fine-tuning** | 100% | 最高（过拟合风险大） | 大规模机器人数据 |
| **LoRA** | 0.1%–1% | 接近全量微调 | **最常用**——新任务快速适配 |
| **QLoRA** | 0.1%–1% | 略低于 LoRA | 消费级 GPU（<24GB 显存） |
| **Prompt Tuning** | <0.01% | 基础任务可接受 | 极低资源，多任务切换 |
| **Frozen + Action Head** | 1%–5% | 视任务而定 | 最大程度保留预训练知识 |

#### OpenVLA 的 LoRA 实践

```
OpenVLA-7B 全量参数：7B
LoRA 微调参数：~8M（0.1%）
├── Vision encoder：冻结（已预训练好）
├── LLM backbone：LoRA 注入 attention 层
└── Action head：全量训练
```

**关键经验：**
- LoRA rank = 32–64 即可
- 同时注入 attention query/value 和 MLP down-projection
- 50–100 条高质量演示即可完成新任务适配

#### Test-Time Adaptation

在部署时实时适应环境变化：

| 方法 | 原理 | 典型应用 |
|------|------|---------|
| **视觉域适应** | 在线调整视觉编码器的 BN 统计量 | 光照/背景变化 |
| **动作残差学习** | 学习"基座策略 + 环境相关偏移" | 不同摩擦力/负载 |
| **上下文学习** | 将成功示例作为上下文注入 prompt | 新物体快速适应 |

### 4.5 安全与对齐：VLA 不能有"幻觉"

LLM 的幻觉是文字错误——VLA 的幻觉是**物理灾难**。这催生了专门的 VLA 安全技术：

#### 安全技术全景

| 层级 | 方法 | 机制 |
|------|------|------|
| **数据层** | 安全数据过滤 + 对抗示例训练 | 训练时注入安全约束 |
| **推理层** | 约束解码（Constrained Decoding） | 强制动作在安全范围内（关节限位、力矩上限） |
| **执行层** | 运行时安全监视器 | 独立进程检测异常动作并紧急停止 |
| **对齐层** | RLHF/Constitutional VLA | 用人类偏好训练"安全"的行为策略 |

```
约束解码示例：
    模型输出关节角度 [-180° ~ 180°]
        ↓ 约束层
    角度 ∈ [0°, 90°]  ← 物理限位映射
    力矩 ∈ [0, 5 N·m] ← 安全力矩上限
    速度 ∈ [0, 2 rad/s] ← 平缓运动约束
```

#### Constitutional VLA（2025 探索方向）

将"机器人三定律"编码为 VLA 的训练约束：
- 每次动作输出前，自动检查是否违反安全规则
- 训练时用违反案例的反例做偏好对齐
- 类似于 Anthropic 的 Constitutional AI，但约束对象是**动作空间**

### 4.6 推理加速：大模型上机器人

VLA 推理延迟是部署的核心瓶颈。典型的性能目标：

| 控制层级 | 目标频率 | 目标延迟 | 当前 VLA 瓶颈 |
|---------|---------|---------|-------------|
| 高层规划 | 1–5 Hz | 200–1000ms | ✅ VLM 推理已满足 |
| 技能执行 | 10–50 Hz | 20–100ms | ⚠️ 扩散采样步数限制 |
| 运动控制 | 100–1000 Hz | 1–10ms | ❌ 大模型无法达标 |

#### 加速技术矩阵

| 技术 | 加速倍数 | 精度损失 | 原理 |
|------|---------|---------|------|
| **INT8/INT4 量化** | 2×–4× | <1% | 权重/激活低精度 |
| **知识蒸馏** | 2×–10× | 1%–3% | 大模型教小模型 |
| **Speculative Decoding** | 1.5×–3× | 0% | 小模型预测 + 大模型验证 |
| **TensorRT/ONNX 编译** | 1.5×–3× | <0.5% | 计算图优化 + 算子融合 |
| **Flow Matching 减少步数** | 5×–10× | 1%–2% | 少步数等效采样 |
| **KV-Cache 复用** | 1.2×–1.5× | 0% | 历史帧缓存共享 |

#### 实战部署方案：Edge VLA

```
云端（离线训练）
    大模型预训练 + LoRA 微调
        ↓ 蒸馏
边缘设备（在线推理）
    学生模型（<1B）
    + INT8 量化
    + TensorRT 优化
    → 推理延迟 ~15ms（Jetson AGX Orin）
```

> 典型方案：OpenVLA-7B 教师 → **TinyVLA-300M** 学生 → INT4 量化 → 部署在 Jetson Orin，推理延迟从 500ms 降至 25ms。

## 五、与你正在做的 GraspAnything 的关系

GraspAnything 本质是一个**轻量 VLA 系统**：

```
Vision:  Grounding DINO + SAM → 开放词汇物体检测与分割
Language: 自然语言描述目标物体
Action:  robosuite 抓取执行
```

在 VLA 全景图中的位置：

```
完整 VLA（RT-2 / Pi0）:  端到端感知→推理→动作
        ↑
GraspAnything:           模块化 VLA（检测→分割→抓取规划→执行）
        ↑
传统方法:                预编程技能 + 固定物体集合
```

> 它是理解 VLA 全栈的最佳入门——从简单系统出发，逐步理解 RT-2、Pi0 等大模型的设计动机：为什么需要端到端？为什么需要海量数据？为什么扩散比回归好？
