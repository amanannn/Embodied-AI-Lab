## 一、定义与核心特征

基础模型（Foundation Model）是斯坦福 HAI 在 2021 年提出的概念：**在大规模、多样化数据上预训练，可适配广泛下游任务的通用模型**。

| 特征 | 说明 |
|------|------|
| **大规模预训练** | 在海量无标注数据上完成自监督/半监督学习 |
| **涌现能力** | 规模突破临界点后出现小模型不具备的能力 |
| **多任务迁移** | 同一模型可适配分类、生成、推理、翻译等 |
| **多模态融合** | 文本、图像、语音、视频的统一建模 |

> 基础模型的范式转换：从"为每个任务训练专用模型"到"训练一个通用底座，再适配所有任务"。

## 二、基础模型演进

| 时期 | 阶段 | 代表 | 关键变化 |
|------|------|------|---------|
| 2018-2019 | 预训练语言模型 | BERT, GPT-1/2 | 证明了"预训练 + 微调"范式的有效性 |
| 2020-2021 | 大规模语言模型 | GPT-3 (1750亿) | 上下文学习取代微调，涌现能力被观察到 |
| 2021-2022 | 多模态基础模型 | CLIP, DALL-E, Stable Diffusion | 视觉和语言进入同一语义空间 |
| 2023 | 统一多模态 | GPT-4V, Gemini | 视觉理解 + 语言推理 + 代码生成一体化 |
| 2024+ | 具身基础模型 | RT-2, Octo, Pi0 | 基础模型从屏幕走向物理世界 |

## 三、基础模型的三个层次

### 3.1 语言基础模型

| 模型 | 参数规模 | 核心能力 | 范式 |
|------|---------|---------|------|
| BERT | 110M / 340M | 双向上下文理解 | 预训练 + 微调 |
| GPT-3 | 175B | 上下文学习、自然语言生成 | Prompt + Few-shot |
| GPT-4 | ~1.8T | 多模态理解、复杂推理、代码生成 | 指令遵循 + Agent |
| LLaMA 系列 | 7B-405B | 开源生态、社区驱动创新 | 开放权重 |

### 3.2 视觉基础模型

| 模型 | 核心能力 | 关键创新 |
|------|---------|---------|
| ViT | 图像分类骨干 | Patch 化——图像也能用 Transformer 处理 |
| MAE | 自监督视觉预训练 | 75% 极高掩码率 + 非对称编解码器 |
| SAM | 通用图像分割 | Promptable——点/框/文本即可分割任意物体 |
| DINO v2 | 强视觉表征 | 自监督学习产生媲美监督学习的特征 |

### 3.3 多模态基础模型

| 模型 | 能力 | 架构 |
|------|------|------|
| CLIP | 图文匹配 + Zero-shot 分类 | 双塔（ViT + Transformer）→ 对比学习 |
| BLIP-2 | 图文理解 + 生成 | Q-Former 桥接视觉编码器和 LLM |
| LLaVA | 多模态对话 | ViT + LLM + 投影层 |
| GPT-4V | 视觉推理 + 对话 | 端到端多模态 Transformer |

## 四、为什么基础模型重要？

| 意义 | 说明 |
|------|------|
| **统一范式** | NLP + CV + 语音 → 同一套架构和方法论 |
| **降低门槛** | 一个人 + 一个预训练模型 = 之前一个团队的产出 |
| **加速研究** | 不再从零训练，在基础模型上微调或 prompt |
| **催化应用** | ChatGPT、Midjourney、Copilot 都基于此 |

> 基础模型不是"又一个新模型"，而是 AI 研发模式的根本转变——从"造轮子"到"用轮子"。

---

## 五、基础模型的关键技术

| 技术 | 作用 | 代表 |
|------|------|------|
| **Transformer** | 基础模型的统一架构 | Self-Attention + FFN |
| **自监督学习** | 摆脱标注依赖 | MLM、对比学习、MAE |
| **Scaling Law** | 指导模型和数据规模的扩展 | Kaplan et al., 2020 |
| **指令微调** | 让模型遵循人类意图 | InstructGPT, FLAN |
| **RLHF / DPO** | 对齐人类价值观 | ChatGPT, Claude |
| **MoE** | 稀疏激活扩大容量而不增加推理成本 | Mixtral, DeepSeek-V2 |

## 六、基础模型与具身智能

基础模型正在从纯软件世界走向物理世界：

| 方向 | 代表工作 | 基础模型在其中扮演什么角色 |
|------|---------|--------------------------|
| 视觉感知 | SAM, Grounding DINO | 通用物体定位和分割 |
| 任务理解 | GPT-4V, LLaVA | 自然语言指令→任务规划 |
| 动作生成 | RT-2, Octo, Pi0 | 视觉+语言→机器人动作 |
| 通用策略 | GATO, RT-X | 多任务、多本体统一模型 |

> 具身智能的终极形态 = 视觉基础模型（看懂世界）+ 语言基础模型（理解任务）+ 动作基础模型（执行动作）。
