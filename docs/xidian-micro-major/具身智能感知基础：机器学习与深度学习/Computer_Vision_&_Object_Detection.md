## 一、目标检测：回答"物体在哪里"

分类回答"图中是什么"，检测进一步要求**找到物体的位置**。

| 任务 | 输出粒度 | 输出形式 | 代表方法 |
|------|---------|---------|---------|
| 图像分类 | 图像级 | "这是一张猫的照片" | ResNet, ViT |
| **目标检测** | 区域级 | "猫在 (x1,y1)→(x2,y2)" | R-CNN, YOLO, DETR |
| 语义分割 | 像素级 | "每个像素属于猫" | FCN, DeepLab |
| 实例分割 | 像素级+实例 | "这是第一只猫，这是第二只" | Mask R-CNN, SAM |

---

## 二、检测的核心挑战

| 挑战 | 说明 | 应对 |
|------|------|------|
| **定位** | 物体可出现在图像任意位置 | 密集候选框 + 回归精修 |
| **尺度变化** | 同一类物体大小悬殊 | 多尺度特征金字塔（FPN） |
| **遮挡** | 物体可能部分被遮挡 | 数据增强 + 上下文推理 |
| **速度** | 实时检测需要毫秒级 | 单阶段（YOLO） |
| **小目标** | 远处小物体信息量少 | 高分辨率 + 特征融合 |

## 三、检测方法演进

### 3.1 两阶段检测（Two-Stage）

```
输入 → 候选区域生成 → 区域分类 + 位置精修 → 输出框
```

| 模型 | 年份 | 核心创新 | 特点 |
|------|------|---------|------|
| **R-CNN** | 2014 | Selective Search → CNN 分类 | 慢但开创范式 |
| **Fast R-CNN** | 2015 | 全图一次 CNN → RoI Pooling | 比 R-CNN 快 25× |
| **Faster R-CNN** | 2015 | RPN 替代 Selective Search | 端到端可训练 |
| **Mask R-CNN** | 2017 | + 分割分支 | 检测 + 实例分割一体 |

#### Faster R-CNN 四组件

| 组件 | 功能 |
|------|------|
| **Backbone** | CNN 提取图像特征图 |
| **RPN** | 滑动 anchor → "这里有物体吗？" + 粗修位置 |
| **RoI Pooling** | 不同尺寸候选区统一到固定尺寸 |
| **Head** | 分类（哪类？）+ 回归（精确位置？） |

### 3.2 单阶段检测（One-Stage）

```
输入 → 密集预测（同时分类+回归） → 输出框
```

| 模型 | 年份 | 核心创新 | 特点 |
|------|------|---------|------|
| **YOLOv1** | 2016 | S×S 网格 + 每格预测 B 个框 | 45 FPS |
| **SSD** | 2016 | 多尺度特征层同时检测 | 兼顾大小目标 |
| **RetinaNet** | 2017 | Focal Loss 解决正负样本不均衡 | 精度匹敌两阶段 |
| **YOLOv3→v10** | 2018-2024 | FPN/锚点优化/数据增强 | 工业界事实标准 |

> 单阶段的哲学：放弃"先粗筛再精修"，直接端到端预测。用速度换精度，差距已很小。

### 3.3 Transformer 检测

| 模型 | 年份 | 核心创新 |
|------|------|---------|
| **DETR** | 2020 | CNN → Transformer → 二分图匹配——无需 NMS |
| **Deformable DETR** | 2021 | 可变形注意力，收敛快 10× |
| **Grounding DINO** | 2023 | 文本条件检测——"红色的杯子"直接定位 |

> DETR 的范式转变：从"anchor + NMS 后处理"变成"集合预测"——不再需要手工锚点和去重。

## 四、对比总结

| 维度 | 两阶段（Faster R-CNN） | 单阶段（YOLO） | Transformer（DETR） |
|------|----------------------|---------------|-------------------|
| 精度 | ⭐⭐⭐ | ⭐⭐→⭐⭐⭐ | ⭐⭐⭐ |
| 速度 | ⭐ | ⭐⭐⭐ | ⭐⭐ |
| 小目标 | ⭐⭐⭐ | ⭐⭐ | ⭐⭐ |
| 后处理 | NMS | NMS | 无需 NMS |
| 部署友好 | 低 | 极高 | 中 |

## 五、与具身智能的关系

| 应用 | 检测的职责 |
|------|-----------|
| **物体抓取** | "我要抓的东西在哪？" |
| **导航避障** | "前面有椅子，绕开" |
| **场景理解** | 检测物体 + 空间关系 |
| **人机交互** | 检测手势、姿态、动作 |

> 检测是具身智能的"眼睛"——几乎所有物理任务第一步都是"找到目标"。
