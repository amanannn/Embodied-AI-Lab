## 一、逐际动力公司概览

逐际动力（LimX Dynamics）是具身智能控制领域的头部企业。

### 1.1 产品线

#### TRON 1——首款多形态双足机器人

| 特性 | 说明 |
|------|------|
| **形态** | 双点足、双足、双轮足 |
| **开发接口** | 全开放 SDK 和底层硬件接口，实用外设接口 |
| **开发语言** | 支持仅用 Python 进行全流程开发 |
| **仿真平台** | NVIDIA Isaac、MuJoCo、Gazebo 等 |

#### TRON 2——多形态具身机器人

| 特性 | 说明 |
|------|------|
| **形态** | 双臂、双轮足、双足 |
| **数据闭环** | 一站式完成数采、清洗、标注、训练、推理与任务管理 |

| **框架支持** | ROS2 + ROS |
| **接口** | 全开放 SDK，全开放上层与底层接口 |
| **仿真** | URDF 提升 Sim2Real 效果，支持主流仿真平台 |

#### LimX Oli——全尺寸通用人形机器人

| 维度 | 规格 |
|------|------|
| **自由度** | 全尺寸机身 31 自由度 |
| **末端执行器** | 多款可选 |

| **视觉** | 头部 + 胸前双 Intel® RealSense™ D435i 深度相机 |
| **IMU** | 自研 6 轴 IMU |
| **接口** | USB 3.0/3.2、RJ45 以太网、24V 5A 与 12V 5A 供电 |

#### LimX Luna——AI 任务编辑器

> 搭载行业首个 AI 任务编辑器——只需描述场景需求，AI 即可自动组合舞蹈、动作、视觉与语音等能力。

## 二、FluxVLA Engine——全栈端到端 VLA 工程平台

面向具身智能应用的全栈端到端工程平台，打通从数据到真机部署的完整工程闭环，为产学研提供标准化底座，显著降低 VLA 研发工程门槛。

### 2.1 核心设计原则

统一配置、标准化接口、模块解耦、可部署性。

### 2.2 统一模块化 VLA 主干

所有 Model 均继承 `BaseVLA`——Vision Encoder、Language Encoder、向 LLM 嵌入空间的 Projection Layer 与 ActionPrediction Head——可在以下模型间切换而无需重写整套训练流程：

OpenVLA、LlavaVLA、GR00T、Pi0、Pi0.5

### 2.3 骨干网络覆盖

| 类别 | 支持模型 |
|------|---------|
| **Language Model** | LLaMA、Gemma、Qwen 等系列 |
| **视觉** | DinoSigLIP（DINO + SigLIP） |
| **视觉-语言** | PaliGemma、QwenVL |

### 2.4 数据与训练

| 维度 | 能力 |
|------|------|
| **数据格式** | 原生支持 Parquet 与 RLDS Pipeline，面向异构数据的 Multi-dataset Mixed Training |
| **分布式训练** | 面向大规模任务的 FSDP 与 DDP |
| **实用训练栈** | LoRA、Mixed Precision（AMP）、Checkpoint Resume、训练结束后自动 Evaluation |

### 2.5 从 Benchmark 到真机

| 阶段 | 能力 |
|------|------|
| **评估** | 多 GPU Evaluation、LIBERO（含无光线追踪环境，如 A100） |
| **推理** | Real-robot Inference Script、跳过加载完整 Pretrained Weight 的 Inference Mode 以节省显存 |

### 2.6 工程架构

| 模块 | 内容 |
|------|------|
| **`fluxvla/`** | Model（VLA、Backbone、Head、Projector）、Dataset、Transform、Tokenizer、Engine、Optimizer、Collator |
| **`configs/`** | 按 Model Family 组织（openvla、llava、gr00t、pi0、pi05） |
| **`scripts/`** | 串联 Training、Evaluation 与 Real-robot Inference |

端到端训练流程：Parquet/RLDS Loading → Transform + Batch Assembly → Forward → Action Loss → Backward。

### 2.7 工具链

| 工具 | 版本/说明 |
|------|----------|
| PyTorch | 2.6 |
| Hugging Face Transformers | 4.53.x |
| Flash Attention | 2.5.x |
| TensorFlow | 用于 RLDS |
| LIBERO | 操作、多任务学习、迁移学习与 VLA 研究迭代 |

### 2.8 路线图

更多视觉/VLM 骨干与 VLA 方法、VLM 或思维链（CoT）数据训练、Isaac Sim 集成、更完善的日志能力。

## 四、具身智能行业发展现状

### 4.1 落地现状

> 真正能落地的方向很少。现在有明确需求的方向：负载方向、舞蹈方向。其他方面还急需探索。

| 现状 | 说明 |
|------|------|
| **硬件限制** | 不能长时间工作，关节过热 |
| **场景适配** | 不是根据场景设计的 |
| **潜在市场** | 农业、单调/无聊/危险的场景、外卖、网约车——可以落地但需要时间 |
| **高校市场** | 卖高校科研——包容度非常高 |

### 4.2 行业反思

> 草台班子，期待值太高。

- 行业发展趋势不一定是单调递增的——机器人的发展可能跟深度学习类似，有涨有跌
- 在涨的阶段可以吃到行业红利，但在跌的阶段该怎么办？

### 4.3 人才困境

| 问题 | 现状 |
|------|------|
| **行业最缺** | 正规专业的从业者 |
| **普通学生的困境** | 没有那么多资源的高校学生、不那么优秀的学生怎么真正进入行业？ |
| **技术栈** | 杂，学习路线陡峭 |
| **本科生门槛** | 高 |

> 结论：需要争夺资源，需要卷，争夺真正高成本的资源。兴趣驱动，多接触开源项目。
