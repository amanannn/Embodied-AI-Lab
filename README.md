# Embodied AI Lab — 具身智能实验课

> 从零构建，逐行理解。每个实验让你亲手写出一个会感知、会决策、会行动的机器人模块。

[![Python](https://img.shields.io/badge/Python-3.10%2B-blue?logo=python)](https://www.python.org/)
[![License](https://img.shields.io/badge/License-MIT-green)](./LICENSE)

---

## 这是什么

一本**实验驱动**的具身智能入门课。每个实验是一个独立、可运行、有可视化的 Python 项目——你写代码，你看到结果，你理解原理。

市面上大多数具身智能学习资源是知识索引和概念讲解——告诉你领域里有什么、去哪学。这门课做的是另一件事：**带你亲手做出来。**

## 实验体系

```
S1 全栈自主导航 + ROS2 对比分析
│
├── 感知线 (3个): 01 KF ✅ → 02 MCL → 03 EKF-SLAM
├── 行动线 (3个): 04 仿真世界 → 05 PID → 06 运动学
├── 规划线 (2个): 07 搜索采样 → 08 轨迹优化
├── 学习线 (3个): 09 Q-Learning → 10 DRL → 11 模仿学习
├── VLA 线 (1个): 12 视觉-语言 Grounding
└── 系统线 (1个): S1 全栈导航
```

### 感知线 — "传感器看到了什么？"

| # | 实验 | 你构建什么 | 状态 |
|---|------|-----------|:----:|
| 01 | **卡尔曼滤波：从零到精通** | KF → EKF → UKF → 粒子滤波，四种滤波器+动画可视化 | ✅ |
| 02 | **粒子滤波定位 (MCL)** | 栅格地图 + 激光雷达模型 + 粒子从全图均匀到收敛在真实位置 | 🚧 |
| 03 | **EKF-SLAM** | 扩展现有 EKF，状态向量加入地标，机器人边走边建图 | 🚧 |

### 行动线 — "机器人身体怎么动？"

| # | 实验 | 你构建什么 | 状态 |
|---|------|-----------|:----:|
| 04 | **2D 仿真世界** | 差速驱动机器人 + 传感器噪声 + 障碍物碰撞 —— 全课程共享基础设施 | 🚧 |
| 05 | **PID 控制实验室** | P→PI→PID，滑块拖拽实时看 Kp/Ki/Kd，机器人循迹 | 🚧 |
| 06 | **机械臂运动学** | 2→6 DOF，正/逆运动学，拖拽末端实时解算关节角 | 🚧 |

### 规划线 — "机器人怎么决策？"

| # | 实验 | 你构建什么 | 状态 |
|---|------|-----------|:----:|
| 07 | **搜索与采样规划** | Dijkstra→A*→RRT→RRT*，搜索树和随机树在屏幕上生长 | 🚧 |
| 08 | **轨迹优化** | 最小 jerk/snap 平滑，折线路径→光滑曲线 | 🚧 |

### 学习线 — "机器人怎么自己学会？"

| # | 实验 | 你构建什么 | 状态 |
|---|------|-----------|:----:|
| 09 | **表格 Q-Learning** | 5×5 网格世界，终端彩色 Q 表热力图，智能体从乱撞到最优 | 🚧 |
| 10 | **深度强化学习** | DQN→PPO，在 04 仿真器里训练导航策略，TensorBoard | 🚧 |
| 11 | **模仿学习** | 手动遥控收集演示 → 行为克隆 → DAgger，策略从模仿到超越 | 🚧 |

### VLA + 系统收尾

## 实验依赖关系

```
01(KF) [done] --> 02(MCL)
  |
  +--> 04(仿真器) --> 05(PID) --> 07(规划) --> 08(轨迹优化)
  |       |              |              |
  |       +--> 09(QLearn) --> 10(DRL) --> 11(模仿学习)
  |       |
  |       +--> 06(运动学)
  |       |
  |       +--> 12(VLA Grounding)
  |
  +--> 03(EKF-SLAM)
                          |
                          v
                        S1(全栈导航)
```

**04（仿真世界）是关键节点**——它被几乎所有后续实验共享。建好它，后续实验构建成本减半。

## 每个实验的标准结构

```
experiments/XX-experiment-name/
├── tutorial.md          # 教程正文：问题→直觉→代码→公式→实验
├── requirements.txt     # numpy + matplotlib（尽量零额外依赖）
├── src/                 # 核心实现，可直接 import
│   └── __init__.py
├── demos/               # 可运行的 demo 脚本
│   └── demo.py
├── utils/               # 可视化和辅助工具
│   └── visualization.py
└── output/              # 生成的图片/动画
```

**教学循环**：动手构建 → 观察行为 → 理解原理 → 调参实验 → 形成直觉。

## 建设优先级

| 批次 | 实验 | 理由 |
|------|------|------|
| 1 | 04 仿真世界 + 05 PID | 共享基础设施 + 补齐"行动"，和 KF 构成最小闭环 |
| 2 | 02 MCL + 07 规划 | 复用 KF 代码，低成本高价值，可视化王者 |
| 3 | 03 EKF-SLAM + 09 Q-Learning | 破 SLAM 恐惧 + 拉人气 |
| 4 | 06 运动学 + 08 轨迹优化 + 10 DRL | 解锁机械臂和深度 RL |
| 5 | 11 模仿学习 + 12 VLA Grounding | 前沿话题 |
| 6 | S1 全栈导航 | 收尾大项目 |
